## 物件導向的核心概念

在進入程式碼前，先理解三個關鍵比喻：

| 概念 | 生活比喻 | Java 術語 |
|------|----------|-----------|
| 類別 (Class) | 「腳踏車」的設計圖 | `class` |
| 物件 (Object) | 根據設計圖造出的某一台腳踏車 | `new` |
| 方法 (Method) | 腳踏車的功能（加速、煞車）| `void upSpeed()` |
| 屬性/欄位 (Field) | 腳踏車的狀態（速度、顏色）| `double speed` |

## 第 20 課 — Brad20：使用物件（Bike 類別初體驗）

**先看 Bike 類別的設計（apis/Bike.java）：**

```java
package tw.brad.apis;

public class Bike {
    // 欄位：物件的「狀態」，用 protected 讓子類別可以存取
    protected double speed;
    
    /*
     * 存取修飾子（決定誰可以看到/使用）：
     * private   → 只有本類別
     * (沒寫)    → 同一個 package
     * protected → 子類別 or 同一個 package
     * public    → 全世界都可以用
     */
    
    // 加速：回傳 this（讓呼叫可以串接）
    public Bike upSpeed() {
        speed = speed < 1 ? 1 : speed * 1.4;
        return this;  // 回傳自己，支援串接呼叫
    }
    
    public Bike downSpeed() {
        speed = speed < 1 ? 0 : speed * 0.7;
        return this;
    }
    
    public double getSpeed() {
        return speed;
    }
    
    @Override
    public String toString() {
        return String.format("Speed: %f", speed);  // 轉成字串時的格式
    }
}
```

**Brad20 使用 Bike：**

```java
public class Brad20 {
    public static void main(String[] args) {
        Bike b1 = new Bike();  // 建立第一台腳踏車物件
        Bike b2 = new Bike();  // 建立第二台腳踏車物件（完全獨立！）
        
        System.out.println(b1.getSpeed()); // 0.0（初始速度）
        
        b1.upSpeed(); b1.upSpeed();
        b1.upSpeed(); b1.upSpeed();
        System.out.println(b1.getSpeed()); // 加速4次後的速度
        
        // b1 和 b2 是獨立物件，互不影響
        b2.upSpeed(); b2.upSpeed(); b2.upSpeed();
        b2.upSpeed(); b2.upSpeed(); b2.upSpeed();
        b2.downSpeed(); b2.downSpeed();
        System.out.println(b2.getSpeed());
    }
}
```

## 第 21 課 — Brad21：繼承（Scooter 繼承 Bike）

```java
// Scooter.java — Scooter「是一種」Bike（is-a 關係）
public class Scooter extends Bike {  // extends = 繼承
    private int gear;       // 新增：排檔
    private String color;   // 新增：顏色
    
    public Scooter() {
        color = "yellow";   // 預設顏色
    }
    
    // @Override：覆寫父類別的方法（改變行為）
    // Scooter 的加速會考慮排檔數
    @Override
    public Scooter upSpeed() {
        speed = speed < 1 ? 1 : speed * 1.8 * gear;
        return this;
    }
    
    // Overload（多載）：同名但參數不同的方法
    public void upSpeed(int gear) {
        changeGear(gear);
        speed = speed < 1 ? 1 : speed * 1.8 * gear;
    }
}
```

```java
// Brad21 使用 Scooter
public class Brad21 {
    public static void main(String[] args) {
        Scooter s1 = new Scooter();
        s1.changeGear(1);           // 換到一檔
        s1.upSpeed(); s1.upSpeed(); s1.upSpeed();
        s1.changeGear(4);           // 換到四檔
        s1.upSpeed(); s1.upSpeed(); s1.upSpeed();
        s1.downSpeed();             // 繼承自 Bike 的方法，Scooter 也有！
        System.out.println(s1.getSpeed());
        System.out.println(s1.getColor()); // yellow
    }
}
```

**Override vs Overload 比較：**
| 特性 | Override（覆寫）| Overload（多載）|
|------|----------------|----------------|
| 方法名稱 | 相同 | 相同 |
| 參數 | **相同** | **不同** |
| 發生位置 | 父子類別之間 | 同一個類別 |
| 目的 | 改變繼承的行為 | 提供多種呼叫方式 |

## 第 23 課 — Brad23：方法多載（Overloading）

```java
class Brad231 {
    void m1() { System.out.println("m1()"); }
    // void m1(byte a) {...}   ← 如果有這個，呼叫 m1(12,12) 會模糊
    void m1(int a) { System.out.println("m1(int)"); }
    void m1(byte a, int b) { System.out.println("m1(byte,int)"); }
    void m1(int a, byte b) { System.out.println("m1(int,byte)"); }
    // ↑ 同名方法，但參數不同 → Java 靠參數型別/數量來區分
}
```

**多載的規則：**
1. 方法名稱相同
2. 參數的**數量**或**型別**必須不同
3. 只有回傳型別不同**不算**多載

## 第 24 課 — Brad24：建構子與 super()

```java
/*
 * 建構子三個規則：
 * 1. 每個類別一定有建構子（沒寫的話 Java 自動給一個無參數建構子）
 * 2. 建構子的第一行一定是呼叫 super() 初始化父類別
 * 3. 建構子不會被繼承
 */
class Brad241 extends Object {
    Brad241(int a) {
        super();  // 明確呼叫 Object 的建構子（不寫也會自動加）
        System.out.println("Brad241(int)");
    }
    Brad241(String a) {
        System.out.println("Brad241(String)");
    }
}

class Brad242 extends Brad241 {
    Brad242() {
        super("Brad");  // 必須指定呼叫哪個父建構子（因為 Brad241 沒有無參建構子）
        System.out.println("Brad242()");
    }
}
```

## 第 25 ~ 27 課 — Brad25/26/27：String 字串

```java
// Brad25：String 的多種建構方式
public class Brad25 {
    public static void main(String[] args) {
        String s1 = new String();          // 空字串
        byte[] b1 = {97, 98, 99, 100};     // a, b, c, d 的 ASCII
        String s2 = new String(b1);         // "abcd"
        String s3 = new String(b1, 1, 2);  // 從索引1取2個 = "bc"
        System.out.println(s2);  // abcd
        System.out.println(s3);  // bc
        System.out.println("Brad".charAt(0));  // 'B'
    }
}

// Brad26：== 和 equals() 的差異（重要！）
public class Brad26 {
    public static void main(String[] args) {
        String s1 = new String("Brad");  // 在堆積記憶體建立新物件
        String s2 = new String("Brad");  // 又建立一個新物件
        String s3 = "Brad";  // 從字串池 (String Pool) 取
        String s4 = "Brad";  // 也從字串池取，s3 和 s4 是同一個物件
        
        System.out.println(s1 == s2);   // false（不同物件）
        System.out.println(s3 == s4);   // true（同一個池中的物件）
        System.out.println(s1 == s3);   // false（堆積 vs 池）
        
        // equals() 比較的是「內容」，而不是記憶體位址
        System.out.println(s1.equals(s2));  // true（內容相同）
        System.out.println(s2.equals(s3));  // true
    }
}

// Brad27：String 是不可變的（Immutable）
public class Brad27 {
    public static void main(String[] args) {
        String s1 = "Brad";
        String s2 = s1.concat("Chao");  // concat 產生一個「新」字串
        System.out.println(s1);  // Brad（原字串沒有改變！）
        System.out.println(s2);  // BradChao
    }
}
```

**⚠️ String 最重要的概念：**
- 字串比較**一定要用 `.equals()`**，不要用 `==`
- String 物件是**不可變的**，所有修改都會產生新字串

## 第 29 課 — Brad29：this() 建構子鏈

```java
class Brad292 extends Brad291 {
    Brad292() {
        this(1);  // 呼叫同類別的另一個建構子 Brad292(int)
    }
    Brad292(int a) {

    }
    Brad292(double b) {
        // 最終執行的建構子
    }
}
```

**`this()` vs `super()`：**
- `this(...)` → 呼叫**同一個類別**的另一個建構子
- `super(...)` → 呼叫**父類別**的建構子
- 兩者都只能放在建構子的第一行，且只能選一個

## 第 30 課 — Brad30：靜態成員 vs 實例成員

```java
class Brad303 {
    int a;       // 實例變數：每個物件各自有一份
    static int b; // 靜態變數：所有物件共用同一份！
    
    Brad303() {
        a++;  // 每個物件自己的 a 從 0 變 1
        b++;  // 所有物件共用的 b 累加
    }
}

// 執行結果：
// 建立3個 Brad303 物件後：
// 每個物件的 a 都是 1（各自計算）
// 全域的 b 變成 3（累加了3次）
```

**靜態 vs 實例的差異：**
| 特性 | 靜態 (static) | 實例 |
|------|--------------|------|
| 屬於 | 類別 | 物件 |
| 存取方式 | `類別名稱.成員` | `物件.成員` |
| 共享 | 所有物件共用 | 每個物件獨立 |
| 生命週期 | 程式執行期間 | 物件存活期間 |

## 第 31 課 — Brad31：多型（Polymorphism）

```java
// 繼承結構：Brad312, Brad313 都繼承 Brad311
class Brad311 { void m1() {...} }
class Brad312 extends Brad311 { void m1() {...} void m2() {...} }
class Brad313 extends Brad311 { void m1() {...} void m3() {...} }

public class Brad31 {
    public static void main(String[] args) {
        // 向上轉型（Upcasting）：子類別物件存到父類別變數
        Brad311 obj2 = new Brad312();  // OK！Brad312 is-a Brad311
        obj2.m1();  // 執行的是 Brad312 的 m1()（動態綁定）
        
        // 向下轉型（Downcasting）：需要 (型別) 強制轉換
        Brad312 obj3 = (Brad312)obj2;  // 必須確定 obj2 確實是 Brad312
        obj3.m2();  // 現在可以呼叫 Brad312 特有的 m2()
        
        // 不能把 Brad312 轉成 Brad313（它們不是父子關係）
        // Brad313 obj6 = (Brad313)obj2; ← 這會報 ClassCastException！
    }
    
    // instanceof：安全地檢查物件的真實型別
    static void test(Brad311 car) {
        car.m1();
        if (car instanceof Brad312) {
            ((Brad312)car).m2();
        } else if (car instanceof Brad313) {
            ((Brad313)car).m3();
        }
    }
}
```

## 第 32 課 — Brad32：抽象類別

// abstract class：不能直接 new，必須被繼承後使用
abstract class Brad321 {

        System.out.println("匿名實作");
        return 0;
    }
    @Override
    void m2() {
        System.out.println("匿名 m2");
    }
};

## 第 33 課 — Brad33：介面（Interface）

```java
// interface：純粹定義「行為規格」，沒有實作
interface Brad331 {
    void m1();
    void m2();
}
interface Brad333 {
    void m3();
    void m4();
}

// 一個類別可以「繼承」一個類別，但可以「實作」多個介面
class Brad332 extends Object implements Brad331, Brad333 {
    public void m1() {}
    public void m2() {}
    public void m3() {}
    public void m4() {}
}

// 介面也可以繼承介面（甚至繼承多個）
interface Brad334 extends Brad331, Brad333 {
    void m1234();
}
```

**抽象類別 vs 介面：**
| 特性 | 抽象類別 | 介面 |
|------|---------|------|
| 關鍵字 | `abstract class` | `interface` |
| 繼承 | 只能繼承一個 | 可以實作多個 |
| 方法 | 可以有實作 | 預設都是抽象（Java 8+ 可有 `default`）|
| 欄位 | 可以有實例變數 | 只能有常數 |

## 第 34 ~ 40 課 — 集合框架（Collections）

### Brad34：HashSet（無序、不重複）

```java
import java.util.*;

public class Brad34 {
    public static void main(String[] args) {
        // Integer 是 int 的包裝類別（Wrapper Class）
        Integer a = 10;
        System.out.println(a.intValue());  // 取出 int 值
        
        // HashSet：不允許重複，無固定順序
        Set set = new HashSet();
        set.add("Brad");
        set.add(123);    // int 自動裝箱（auto-boxing）成 Integer
        set.add(12.3);   // double 自動裝箱成 Double
        set.add("Brad"); // 重複的 "Brad" 不會被加入！
        System.out.println(set.size()); // 4（不是5）
        
        // 用 Iterator 遍歷
        Iterator it = set.iterator();
        while (it.hasNext()) {
            Object obj = it.next();
            System.out.println(obj);
        }
        
        // 更簡潔：for-each
        for (Object obj : set) {
            System.out.println(obj);
        }
    }
}
```

## 第 41 ~ 43 課 — 例外處理（Exception Handling）

### Brad41：try-catch

```java
public class Brad41 {
    public static void main(String[] args) {
        int a = 10, b = 3;
        int[] d = {1, 2, 3, 4};
        
        System.out.println("Brad");
        try {
            // 這裡的程式碼如果出錯，會跳到 catch
            int c = a / b;                // 正常，c = 3
            System.out.println(c);
            System.out.println(d[200]);   // ← 出錯！索引超界
        } catch (RuntimeException e) {
            // 捕捉例外並處理
            System.out.println(-1);       // 印出 -1
        }
        System.out.println("Finish");     // 繼續執行
    }
}
```

### Brad42：throws 和 throw（自訂例外拋出）

```java
class Bird {
    int leg;
    
    // throws：宣告「這個方法可能會拋出例外」，呼叫者要處理
    void setLeg(int n) throws IOException {
        if (n >= 0 && n <= 3) {
            leg = n;
        } else {
            throw new FileNotFoundException();  // 主動拋出例外
        }
    }
}

public class Brad42 {
    public static void main(String[] args) {
        Bird b1 = new Bird();
        try {
            b1.setLeg(5);  // 5 超出範圍，會拋出例外
        } catch (IOException e) {
            System.out.println("Err");
        }
        System.out.println(b1.leg);  // 0（因為沒有成功設定）
    }
}
```

### Brad43：finally 區塊

```java
static void m1(int a) {
    try {
        m2(7);  // 呼叫可能拋出例外的方法
        System.out.println("OK");      // 如果沒例外才會執行
    } catch (Exception e) {
        System.out.println("ee");      // 捕捉例外
        return;                         // 提前返回
    } finally {
        System.out.println("ff");      // 不管有沒有例外，一定執行！
    }
    System.out.println("END");         // 如果 catch 裡有 return，這行不會執行
}
```

**例外處理流程：**
```
try {
    可能出錯的程式碼
} catch (例外型別 e) {
    出錯時的處理
} finally {
    不管如何都執行（通常用來關閉資源）
}
```

**例外層次結構：**
- `Throwable`
  - `Error`（系統錯誤，通常不處理）
  - `Exception`
    - `RuntimeException`（執行時才出現，可以不 catch）
      - `NullPointerException`
      - `ArrayIndexOutOfBoundsException`
      - `ClassCastException`
    - 其他 Exception（必須 catch 或 throws）
      - `IOException`
      - `FileNotFoundException`

### 【填充題】

1. 繼承用關鍵字 `______`，實作介面用關鍵字 `______`。

2. 要比較兩個 String 的內容是否相同，應使用 `______` 方法而非 `==`。

3. `ArrayList` 允許重複元素，而 `______` 不允許重複。

4. `HashSet` 和 `TreeSet` 的差別是：TreeSet 會自動對元素進行 `______`。

5. `try-catch-finally` 中，`______` 區塊不管有沒有例外都一定會執行。

### 【判斷題】

7. 抽象類別可以直接用 `new` 建立物件。`( )`

8. Override 時，子類別方法的存取修飾子可以比父類別更**嚴格**。`( )`

9. `static` 方法可以直接存取同類別的非靜態欄位。`( )`

10. `throw` 用於宣告方法可能拋出例外，`throws` 用於實際拋出例外。`( )`

11. `HashMap` 中的 Key 不能重複，但 Value 可以重複。`( )`

### 【程式題】

**題目 A（基礎）：** 設計一個 `Animal` 類別，有 `name` 欄位和 `speak()` 方法。再設計 `Dog` 和 `Cat` 子類別，覆寫 `speak()` 分別印出「汪汪！」和「喵喵！」。

**題目 B（中等）：** 使用 `ArrayList<String>` 儲存 5 個朋友的名字，然後：
1. 印出所有名字
2. 移除第二個名字
3. 加入一個新名字到最後
4. 再印出一次

**題目 C（中等）：** 設計一個 `Calculator` 類別，使用多載提供三個版本的 `add()` 方法：兩個 `int`、兩個 `double`、以及三個 `int`。

**題目 D（進階）：** 設計一個 `BankAccount` 類別：
- 欄位：帳號（private）、餘額（private）
- 方法：`deposit(double amount)` 存款、`withdraw(double amount)` 提款（餘額不足要拋出例外）、`getBalance()` 查詢餘額
- 提款時若金額超過餘額，拋出 `IllegalArgumentException` 並附上說明訊息

**題目 E（挑戰）：** 設計一個 `Shape` 介面，包含 `getArea()` 和 `getPerimeter()` 方法。再讓 `Circle`（圓形）和 `Rectangle`（長方形）實作這個介面，計算各自的面積與周長。

**填充題：** 1. `extends` / `implements` 2. `.equals()` 3. `HashSet`（或 `TreeSet`）4. 排序 5. `finally` 6. 一個 / 多個

**判斷題：** 7. ✗ 8. ✗（只能相同或更寬鬆）9. ✗（static 方法不能存取 non-static 欄位）10. ✗（`throw` 拋出，`throws` 宣告）11. ✓

**選擇題：** 12. C 13. C 14. C 15. B（add(0,5) 後 list 是 [5,10,20]，get(1) 是 10）
