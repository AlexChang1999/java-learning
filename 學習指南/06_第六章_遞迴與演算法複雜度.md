## 一、遞迴（Recursion）

### 什麼是遞迴？

遞迴就是一個方法**呼叫自己**。聽起來很奇怪，但它能讓某些複雜問題變得極度簡潔。

**遞迴的兩個必要條件：**
1. **終止條件（Base Case）**：什麼時候停止呼叫自己
2. **遞迴關係（Recursive Case）**：怎麼縮小問題規模

### 範例一：計算階乘（n!）

```java
// 一般做法（迴圈）
int factorialLoop(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

// 遞迴做法
int factorial(int n) {
    if (n <= 1) return 1;        // 終止條件：1! = 1
    return n * factorial(n - 1); // 遞迴：n! = n × (n-1)!
}
```

**執行過程（factorial(4)）：**
```
factorial(4)
  = 4 × factorial(3)
          = 3 × factorial(2)
                  = 2 × factorial(1)
                              = 1         ← 觸底，開始回傳
                  = 2 × 1 = 2
          = 3 × 2 = 6
  = 4 × 6 = 24
```

**對應到 Stack 記憶體：**
```
呼叫 factorial(4) 時，Stack 的變化：

factorial(4) ← 新增
factorial(3) ← 新增
factorial(2) ← 新增
factorial(1) ← 新增，觸底，開始回傳
factorial(2) ← 收到 1，計算 2×1=2，回傳 2，移除
factorial(3) ← 收到 2，計算 3×2=6，回傳 6，移除
factorial(4) ← 收到 6，計算 4×6=24，回傳 24，移除
```

### 範例二：費氏數列（Fibonacci）

```java
// F(0)=0, F(1)=1, F(n) = F(n-1) + F(n-2)
int fib(int n) {
    if (n <= 0) return 0;   // 終止條件
    if (n == 1) return 1;   // 終止條件
    return fib(n - 1) + fib(n - 2);  // 遞迴
}
```

**⚠️ 這個遞迴很慢！** 
`fib(5)` 會計算 `fib(3)` 兩次、`fib(2)` 三次，大量重複計算。  
`fib(50)` 就慢到幾乎不能用。這就要談到演算法複雜度了。

### 範例三：二分搜尋（Binary Search）

```java
// 在已排序的陣列中找目標值
// 每次把搜尋範圍砍半
int binarySearch(int[] arr, int target, int left, int right) {
    if (left > right) return -1;  // 找不到

    int mid = (left + right) / 2;  // 取中間

    if (arr[mid] == target) return mid;          // 找到了
    if (arr[mid] > target) return binarySearch(arr, target, left, mid - 1);  // 往左找
    return binarySearch(arr, target, mid + 1, right);                         // 往右找
}
```

## 二、演算法複雜度（Big O Notation）

複雜度是用來描述演算法「當資料量增加時，執行時間如何增長」的方式。  
用 **O(f(n))** 表示，n 是輸入的資料量。

### 常見複雜度排行（由快到慢）

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
 極快   很快        線性   還不錯        慢         極慢
```

### 用圖來感受差距（n=1000時）

```
O(1)       →        1 次操作
O(log n)   →       10 次操作
O(n)       →     1000 次操作
O(n log n) →    10000 次操作
O(n²)      → 1000000 次操作  ← 比 O(n) 慢 1000 倍！
O(2ⁿ)      → 10^301 次       ← 宇宙年齡也算不完
```

### 各複雜度對應的程式碼模式

```java
// O(1)：固定次數，不管 n 多大都一樣快
int getFirst(int[] arr) {
    return arr[0];
}

// O(n)：一層迴圈
int sum(int[] arr) {
    int total = 0;
    for (int x : arr) total += x;  // 跑 n 次
    return total;
}

// O(n²)：雙層巢狀迴圈
void printAllPairs(int[] arr) {
    for (int i = 0; i < arr.length; i++) {          // n 次
        for (int j = 0; j < arr.length; j++) {      // n 次
            System.out.println(arr[i] + "," + arr[j]);
        }
    }
    // 總共 n × n = n² 次
}

// O(log n)：每次把問題砍半（二分搜尋）
// 1000 筆資料最多只需要 10 次（2^10 = 1024）
```

## 三、PokerV1 ~ V4 的演算法分析

### PokerV1：暴力法

```java
// 每次亂數，重複就重試
for (int i = 0; i < 52; i++) {
    int temp = (int)(Math.random() * 52);
    
    boolean isRepeat = false;
    for (int j = 0; j < i; j++) {    // 每次檢查前面所有已填的牌
        if (temp == poker[j]) {
            isRepeat = true; break;
        }
    }
    
    if (!isRepeat) poker[i] = temp;
    else i--;  // 重複就重來
}
```

**問題：** 當只剩最後幾張牌要填時，亂數撞到已有牌的機率極高。
- 填第 51 張時，只有 1/52 的機率沒重複 → 平均要試 52 次
- 填第 50 張時，平均要試 26 次
- 整體複雜度遠超過 O(n²)

### PokerV4（Fisher-Yates 洗牌）：O(n)

```java
int[] poker = new int[52];
for (int i = 0; i < 52; i++) poker[i] = i;  // 先建立有序陣列

// 從後往前，每次和前面隨機一個位置交換
for (int i = 51; i > 0; i--) {
    int r = (int)(Math.random() * (i + 1));  // 0 ~ i 的隨機數
    // 交換 poker[i] 和 poker[r]
    int temp = poker[i];
    poker[i] = poker[r];
    poker[r] = temp;
}
// 每個位置只交換一次 → O(n)
```

**為什麼這樣是公平的？**
- 每個排列出現的機率 = 1/52!（所有排列機率相等）
- 數學證明：第一個位置有 52 種可能，第二個 51 種，... 完全公平

### 速度對比（52 張牌）

```
PokerV1：平均約 2,000~5,000 次亂數呼叫
PokerV4：剛好 51 次交換

如果是 1,000,000 張牌：
PokerV1：約需要 10^13 次操作（幾乎不可能完成）
PokerV4：剛好 999,999 次操作（幾秒內完成）
```

## 四、動態規劃（Dynamic Programming）：消滅重複計算

### 問題：遞迴費氏數列有什麼缺點？

```java
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

呼叫 `fib(6)` 時，計算樹長這樣：

```
                    fib(6)
                 ↙         ↘
            fib(5)          fib(4)
           ↙     ↘          ↙    ↘
       fib(4)  fib(3)   fib(3)  fib(2)
       ↙  ↘    ↙  ↘    ↙  ↘
   fib(3)fib(2)fib(2)fib(1)...
   ...

fib(4) 算了 2 次！
fib(3) 算了 3 次！
fib(2) 算了 5 次！
→ 時間複雜度 O(2ⁿ)，n=50 時根本算不完
```

### 解法一：記憶化遞迴（Top-Down Memoization）

「算過的結果存起來，下次直接查表」

```java
import java.util.HashMap;
import java.util.Map;

// 用 HashMap 當記憶體，key=n，value=已算過的結果
Map<Integer, Long> memo = new HashMap<>();

long fibMemo(int n) {
    if (n <= 1) return n;

    // 如果算過了，直接回傳（不重算）
    if (memo.containsKey(n)) {
        return memo.get(n);  // O(1) 查表，省掉大量遞迴
    }

    // 第一次計算，存起來
    long result = fibMemo(n - 1) + fibMemo(n - 2);
    memo.put(n, result);
    return result;
}

// fibMemo(50) = 12586269025，瞬間完成
// 時間複雜度從 O(2ⁿ) 降到 O(n)
```

### 解法二：打表法（Bottom-Up Tabulation）

「從小問題開始算，把結果填入陣列，最後查結果」

```java
long fibTable(int n) {
    if (n <= 1) return n;

    long[] dp = new long[n + 1];  // dp[i] = fib(i)
    dp[0] = 0;  // base case
    dp[1] = 1;  // base case

    // 從小到大填表，每個值只算一次
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];  // 用已知結果推算
    }

    return dp[n];
}

// 陣列視覺化（n=8）：
// dp: [0, 1, 1, 2, 3, 5, 8, 13, 21]
//      ↑  ↑  ↑  ↑  ↑  ↑  ↑   ↑   ↑
//     f0 f1 f2 f3 f4 f5 f6  f7  f8
```

**進一步優化：只需要記最後兩個值，空間 O(1)**

```java
long fibOptimal(int n) {
    if (n <= 1) return n;
    long prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        long curr = prev1 + prev2;  // 只用到前兩個
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

### 動態規劃的核心思想

```
動態規劃適用的場景：
  1. 問題可以拆成「子問題」
  2. 子問題有重複（重複計算）
  3. 子問題的解可以合併成大問題的解

思考步驟：
  Step 1. 定義「狀態」：dp[i] 代表什麼？
  Step 2. 找「轉移方程式」：dp[i] 怎麼從 dp[i-1]、dp[i-2]... 算出來？
  Step 3. 設定「初始值（base case）」
  Step 4. 決定填表方向（從小到大 or 從大到小）
```

## 五、排序演算法比較

### 為什麼排序很重要？

撮合引擎的訂單簿需要**維持價格排序**：買單從高到低，賣單從低到高。  
選錯排序演算法，高頻交易的延遲可能從 1μs 變成 100μs。

### 常見排序演算法比較表

| 演算法 | 平均複雜度 | 最壞情況 | 空間複雜度 | 穩定? | 適用場景 |
|--------|-----------|---------|------------|-------|----------|
| Bubble Sort | O(n²) | O(n²) | O(1) | ✅ | 只適合教學 |
| Selection Sort | O(n²) | O(n²) | O(1) | ❌ | 只適合教學 |
| **Insertion Sort** | O(n²) | O(n²) | O(1) | ✅ | **n 很小時最快** |
| **Merge Sort** | O(n log n) | O(n log n) | O(n) | ✅ | 需要穩定排序 |
| **Quick Sort** | O(n log n) | O(n²) | O(log n) | ❌ | 實務最常用 |
| **Heap Sort** | O(n log n) | O(n log n) | O(1) | ❌ | 記憶體受限時 |
| Tim Sort | O(n log n) | O(n log n) | O(n) | ✅ | **Java/Python 內建** |

> **穩定排序**：相等元素排序後，相對順序不變。對撮合引擎很重要——同價位先進先出（FIFO）。

### Insertion Sort（插入排序）：小資料的贏家

```java
// 概念：像打牌時整理手牌，每次把新牌插入已排序的區域
void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];   // 準備插入的牌
        int j = i - 1;

        // 往左找到正確位置（把比 key 大的元素往右移）
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];  // 往右移一格
            j--;
        }
        arr[j + 1] = key;  // 插入
    }
}

// 視覺化（排序 [5, 2, 4, 1, 3]）：
// 初始：  [5 | 2, 4, 1, 3]   ← | 左邊已排序，右邊未排序
// i=1：   [2, 5 | 4, 1, 3]   ← 2 插到 5 的前面
// i=2：   [2, 4, 5 | 1, 3]
// i=3：   [1, 2, 4, 5 | 3]
// i=4：   [1, 2, 3, 4, 5]
```

**Insertion Sort 的特殊優點：** 對「幾乎已排序的資料」非常快，接近 O(n)。  
Tim Sort 在小區塊（<64 個元素）時就直接用 Insertion Sort。

### Merge Sort（合併排序）：穩定且效能保證

```java
// 概念：分治法。先拆到最小（1個元素），再兩兩合併
// 合併時保持排序 → 整體有序
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;  // 只有 1 個元素，已排序

    int mid = (left + right) / 2;
    mergeSort(arr, left, mid);      // 排左半
    mergeSort(arr, mid + 1, right); // 排右半
    merge(arr, left, mid, right);   // 合併兩個已排序的半段
}

void merge(int[] arr, int left, int mid, int right) {
    // 建立暫存陣列
    int[] temp = Arrays.copyOfRange(arr, left, right + 1);
    int i = 0, j = mid - left + 1, k = left;

    while (i <= mid - left && j <= right - left) {
        // 穩定排序關鍵：相等時先取左邊（保持原始順序）
        if (temp[i] <= temp[j]) arr[k++] = temp[i++];
        else                    arr[k++] = temp[j++];
    }
    while (i <= mid - left) arr[k++] = temp[i++];
    while (j <= right - left) arr[k++] = temp[j++];
}

// 時間複雜度：永遠是 O(n log n)，最壞情況也一樣
// 缺點：需要 O(n) 的額外空間存暫存陣列
```

### Quick Sort（快速排序）：實務最快的排序

```java
// 概念：選一個基準值（pivot），把比它小的放左邊，大的放右邊，再遞迴
void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;

    int pivotIdx = partition(arr, left, right);  // 分區，回傳 pivot 最終位置
    quickSort(arr, left, pivotIdx - 1);           // 排 pivot 左邊
    quickSort(arr, pivotIdx + 1, right);           // 排 pivot 右邊
}

int partition(int[] arr, int left, int right) {
    int pivot = arr[right];  // 選最右邊的元素當 pivot（也可以隨機選）
    int i = left - 1;        // i 指向已確認「比 pivot 小」的區域末端

    for (int j = left; j < right; j++) {
        if (arr[j] <= pivot) {
            i++;
            // 交換 arr[i] 和 arr[j]（把小的元素移到左邊）
            int tmp = arr[i]; arr[i] = arr[j]; arr[j] = tmp;
        }
    }
    // 把 pivot 放到正確位置（左邊都比它小，右邊都比它大）
    int tmp = arr[i + 1]; arr[i + 1] = arr[right]; arr[right] = tmp;
    return i + 1;
}

// 為什麼實務比 Merge Sort 快？
// 1. 原地排序，不需要額外記憶體
// 2. CPU Cache 友好（連續存取陣列）
// 3. 常數係數較小（內層迴圈簡單）

// 最壞情況（O(n²)）：每次選到最大或最小當 pivot
// → 解決方案：隨機選 pivot，或「三數取中法」
```

### Java 內建的 Arrays.sort() 用哪個？

```java
// Java 的排序策略（超聰明）：
int[] primitiveArray = {5, 2, 8, 1, 9};
Arrays.sort(primitiveArray);
// → 使用 Dual-Pivot QuickSort（雙軸快速排序）
//   平均 O(n log n)，比單軸 QuickSort 快約 20%

Integer[] objectArray = {5, 2, 8, 1, 9};
Arrays.sort(objectArray);
// → 使用 TimSort（Insertion Sort + Merge Sort 混合）
//   穩定排序，對現實資料（通常部分有序）很高效

// 為什麼 primitive 用 QuickSort，Object 用 TimSort？
// primitive 不需要穩定排序（int 相等就是相等，順序無意義）
// Object 需要穩定（equals 相等的物件，使用者可能在乎原始順序）
```

### 撮合引擎的排序需求

```
訂單簿維護兩個有序結構：
  買單：TreeMap<Price, Queue<Order>>   → 價格由高到低（出價最高優先成交）
  賣單：TreeMap<Price, Queue<Order>>   → 價格由低到高（要價最低優先成交）

為什麼用 TreeMap 而不是每次排序？
  TreeMap 底層是紅黑樹，插入/刪除/查詢都是 O(log n)
  每次新訂單進來，插入時自動維持排序，不需要重新排序全部
  如果用 ArrayList 每次 sort → O(n log n)，高頻時慘不忍睹
```

## 六、遞迴 vs 迴圈，怎麼選？

| 情境 | 推薦 | 原因 |
|------|------|------|
| 次數固定或有明確計數 | 迴圈 | 效率高，不佔 Stack |
| 樹狀/巢狀結構（資料夾、XML）| 遞迴 | 程式碼簡潔很多 |
| 分治法（二分搜尋、排序）| 遞迴 | 天然配合 |
| 深度可能很大（>10000層）| 迴圈 | 避免 StackOverflow |

## 延伸問題

**Q1：以下遞迴方法計算 `sum(5)` 的結果是什麼？**
```java
int sum(int n) {
    if (n <= 0) return 0;
    return n + sum(n - 1);
}
```
<details>
<summary>答案</summary>
15（= 5+4+3+2+1+0）。這是計算 1 到 n 的總和。
</details>

**Q2：以下哪段程式碼的時間複雜度是 O(n²)？**
```java
// 程式碼 A
for (int i = 0; i < n; i++) {
    System.out.println(i);
    System.out.println(i * 2);
}

// 程式碼 B
for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {
        System.out.println(i + j);
    }
}
```
<details>
<summary>答案</summary>
程式碼 B 是 O(n²)。雖然內層迴圈不是每次都跑 n 次，但總次數是 n(n+1)/2，Big O 只看最高次項，所以是 O(n²)。程式碼 A 是 O(n)。
</details>

**Q3：寫一個遞迴方法，計算一個整數的各位數字總和。例如 `sumDigits(1234)` = 10。**
<details>
<summary>參考答案</summary>

```java
int sumDigits(int n) {
    if (n < 10) return n;              // 終止條件：個位數
    return n % 10 + sumDigits(n / 10); // 取個位 + 剩餘數字的總和
}
```
</details>
