## 一、遞迴（Recursion）

### 什麼是遞迴？

遞迴就是一個方法**呼叫自己**。聽起來很奇怪，但它能讓某些複雜問題變得極度簡潔。

**遞迴的兩個必要條件：**
1. **終止條件（Base Case）**：什麼時候停止呼叫自己
2. **遞迴關係（Recursive Case）**：怎麼縮小問題規模

### 範例一：計算階乘（n!）

```java
// 一般做法（迴圈）
int factorialLoop(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

// 遞迴做法
int factorial(int n) {
    if (n <= 1) return 1;        // 終止條件：1! = 1
    return n * factorial(n - 1); // 遞迴：n! = n × (n-1)!
}
```

**執行過程（factorial(4)）：**
```
factorial(4)
  = 4 × factorial(3)
          = 3 × factorial(2)
                  = 2 × factorial(1)
                              = 1         ← 觸底，開始回傳
                  = 2 × 1 = 2
          = 3 × 2 = 6
  = 4 × 6 = 24
```

**對應到 Stack 記憶體：**
```
呼叫 factorial(4) 時，Stack 的變化：

factorial(4) ← 新增
factorial(3) ← 新增
factorial(2) ← 新增
factorial(1) ← 新增，觸底，開始回傳
factorial(2) ← 收到 1，計算 2×1=2，回傳 2，移除
factorial(3) ← 收到 2，計算 3×2=6，回傳 6，移除
factorial(4) ← 收到 6，計算 4×6=24，回傳 24，移除
```

### 範例二：費氏數列（Fibonacci）

```java
// F(0)=0, F(1)=1, F(n) = F(n-1) + F(n-2)
int fib(int n) {
    if (n <= 0) return 0;   // 終止條件
    if (n == 1) return 1;   // 終止條件
    return fib(n - 1) + fib(n - 2);  // 遞迴
}
```

**⚠️ 這個遞迴很慢！** 
`fib(5)` 會計算 `fib(3)` 兩次、`fib(2)` 三次，大量重複計算。  
`fib(50)` 就慢到幾乎不能用。這就要談到演算法複雜度了。

### 範例三：二分搜尋（Binary Search）

```java
// 在已排序的陣列中找目標值
// 每次把搜尋範圍砍半
int binarySearch(int[] arr, int target, int left, int right) {
    if (left > right) return -1;  // 找不到

    int mid = (left + right) / 2;  // 取中間

    if (arr[mid] == target) return mid;          // 找到了
    if (arr[mid] > target) return binarySearch(arr, target, left, mid - 1);  // 往左找
    return binarySearch(arr, target, mid + 1, right);                         // 往右找
}
```

## 二、演算法複雜度（Big O Notation）

複雜度是用來描述演算法「當資料量增加時，執行時間如何增長」的方式。  
用 **O(f(n))** 表示，n 是輸入的資料量。

### 常見複雜度排行（由快到慢）

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
 極快   很快        線性   還不錯        慢         極慢
```

### 用圖來感受差距（n=1000時）

```
O(1)       →        1 次操作
O(log n)   →       10 次操作
O(n)       →     1000 次操作
O(n log n) →    10000 次操作
O(n²)      → 1000000 次操作  ← 比 O(n) 慢 1000 倍！
O(2ⁿ)      → 10^301 次       ← 宇宙年齡也算不完
```

### 各複雜度對應的程式碼模式

```java
// O(1)：固定次數，不管 n 多大都一樣快
int getFirst(int[] arr) {
    return arr[0];
}

// O(n)：一層迴圈
int sum(int[] arr) {
    int total = 0;
    for (int x : arr) total += x;  // 跑 n 次
    return total;
}

// O(n²)：雙層巢狀迴圈
void printAllPairs(int[] arr) {
    for (int i = 0; i < arr.length; i++) {          // n 次
        for (int j = 0; j < arr.length; j++) {      // n 次
            System.out.println(arr[i] + "," + arr[j]);
        }
    }
    // 總共 n × n = n² 次
}

// O(log n)：每次把問題砍半（二分搜尋）
// 1000 筆資料最多只需要 10 次（2^10 = 1024）
```

## 三、PokerV1 ~ V4 的演算法分析

### PokerV1：暴力法

```java
// 每次亂數，重複就重試
for (int i = 0; i < 52; i++) {
    int temp = (int)(Math.random() * 52);
    
    boolean isRepeat = false;
    for (int j = 0; j < i; j++) {    // 每次檢查前面所有已填的牌
        if (temp == poker[j]) {
            isRepeat = true; break;
        }
    }
    
    if (!isRepeat) poker[i] = temp;
    else i--;  // 重複就重來
}
```

**問題：** 當只剩最後幾張牌要填時，亂數撞到已有牌的機率極高。
- 填第 51 張時，只有 1/52 的機率沒重複 → 平均要試 52 次
- 填第 50 張時，平均要試 26 次
- 整體複雜度遠超過 O(n²)

### PokerV4（Fisher-Yates 洗牌）：O(n)

```java
int[] poker = new int[52];
for (int i = 0; i < 52; i++) poker[i] = i;  // 先建立有序陣列

// 從後往前，每次和前面隨機一個位置交換
for (int i = 51; i > 0; i--) {
    int r = (int)(Math.random() * (i + 1));  // 0 ~ i 的隨機數
    // 交換 poker[i] 和 poker[r]
    int temp = poker[i];
    poker[i] = poker[r];
    poker[r] = temp;
}
// 每個位置只交換一次 → O(n)
```

**為什麼這樣是公平的？**
- 每個排列出現的機率 = 1/52!（所有排列機率相等）
- 數學證明：第一個位置有 52 種可能，第二個 51 種，... 完全公平

### 速度對比（52 張牌）

```
PokerV1：平均約 2,000~5,000 次亂數呼叫
PokerV4：剛好 51 次交換

如果是 1,000,000 張牌：
PokerV1：約需要 10^13 次操作（幾乎不可能完成）
PokerV4：剛好 999,999 次操作（幾秒內完成）
```

## 四、遞迴 vs 迴圈，怎麼選？

| 情境 | 推薦 | 原因 |
|------|------|------|
| 次數固定或有明確計數 | 迴圈 | 效率高，不佔 Stack |
| 樹狀/巢狀結構（資料夾、XML）| 遞迴 | 程式碼簡潔很多 |
| 分治法（二分搜尋、排序）| 遞迴 | 天然配合 |
| 深度可能很大（>10000層）| 迴圈 | 避免 StackOverflow |

## 本章重點整理

| 概念 | 說明 |
|------|------|
| 遞迴 | 方法呼叫自己，必須有終止條件 |
| Stack 與遞迴 | 每層遞迴都在 Stack 佔一層 |
| Big O | 描述演算法隨 n 增長的速度 |
| O(n²) vs O(n) | 差距隨 n 增大急劇擴大 |
| Fisher-Yates | 公平洗牌的 O(n) 演算法 |

## 延伸問題

**Q1：以下遞迴方法計算 `sum(5)` 的結果是什麼？**
```java
int sum(int n) {
    if (n <= 0) return 0;
    return n + sum(n - 1);
}
```
<details>
<summary>答案</summary>
15（= 5+4+3+2+1+0）。這是計算 1 到 n 的總和。
</details>

**Q2：以下哪段程式碼的時間複雜度是 O(n²)？**
```java
// 程式碼 A
for (int i = 0; i < n; i++) {
    System.out.println(i);
    System.out.println(i * 2);
}

// 程式碼 B
for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {
        System.out.println(i + j);
    }
}
```
<details>
<summary>答案</summary>
程式碼 B 是 O(n²)。雖然內層迴圈不是每次都跑 n 次，但總次數是 n(n+1)/2，Big O 只看最高次項，所以是 O(n²)。程式碼 A 是 O(n)。
</details>

**Q3：寫一個遞迴方法，計算一個整數的各位數字總和。例如 `sumDigits(1234)` = 10。**
<details>
<summary>參考答案</summary>

```java
int sumDigits(int n) {
    if (n < 10) return n;              // 終止條件：個位數
    return n % 10 + sumDigits(n / 10); // 取個位 + 剩餘數字的總和
}
```
</details>
