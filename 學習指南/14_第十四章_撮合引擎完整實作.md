## 一、訂單狀態機

```
        建立訂單
            │
            ▼
         PENDING
            │
     ┌──────┴──────┐
     │             │
     ▼             ▼
  ACTIVE         已立刻取消（IOC/FOK 未成交）
  （在訂單簿中）
     │
  ┌──┼──┐
  │  │  │
  ▼  ▼  ▼
部分  全部 取消
成交  成交
  │   │
  ▼   ▼
PARTIALLY   FILLED    CANCELLED
 _FILLED
  │
（剩餘繼續在訂單簿中等待，或被取消）
```

```java
// OrderStatus.java
public enum OrderStatus {
    PENDING,           // 剛建立，還未加入訂單簿
    ACTIVE,            // 在訂單簿中等待撮合
    PARTIALLY_FILLED,  // 部分成交，仍在訂單簿中
    FILLED,            // 全部成交
    CANCELLED          // 已取消
}
```

```java
// Order.java
public class Order {
    private final String orderId;
    private final String symbol;
    private final String side;     // "BUY" or "SELL"
    private final OrderType type;  // LIMIT, MARKET, IOC, FOK
    private final double price;    // 市價單用 0.0
    private final int originalQty; // 原始數量
    private int remainingQty;      // 剩餘數量（撮合後遞減）
    private OrderStatus status;
    private final long timestamp;  // 用於時間優先排序

    public Order(String orderId, String symbol, String side,
                 OrderType type, double price, int qty) {
        this.orderId = orderId;
        this.symbol = symbol;
        this.side = side;
        this.type = type;
        this.price = price;
        this.originalQty = qty;
        this.remainingQty = qty;
        this.status = OrderStatus.PENDING;
        this.timestamp = System.nanoTime();  // 奈秒精度
    }

    // 撮合時扣減數量
    public void fill(int qty) {
        this.remainingQty -= qty;
        if (this.remainingQty == 0) {
            this.status = OrderStatus.FILLED;
        } else {
            this.status = OrderStatus.PARTIALLY_FILLED;
        }
    }

    // Getters...
    public String getOrderId() { return orderId; }
    public String getSide() { return side; }
    public OrderType getType() { return type; }
    public double getPrice() { return price; }
    public int getRemainingQty() { return remainingQty; }
    public OrderStatus getStatus() { return status; }
    public long getTimestamp() { return timestamp; }
    public void setStatus(OrderStatus status) { this.status = status; }
}
```

## 四、PriceLevel 與訂單簿結構

```java
// PriceLevel.java — 同一價格的所有訂單（時間優先）
import java.util.*;

public class PriceLevel {
    private final double price;
    private final Queue<Order> orders = new LinkedList<>();  // FIFO
    private int totalQty = 0;

    public PriceLevel(double price) {
        this.price = price;
    }

    public void addOrder(Order order) {
        orders.offer(order);
        totalQty += order.getRemainingQty();
    }

    public void removeOrder(Order order) {
        orders.remove(order);
        totalQty -= order.getRemainingQty();
    }

    public Queue<Order> getOrders() { return orders; }
    public double getPrice() { return price; }
    public int getTotalQty() { return totalQty; }
    public boolean isEmpty() { return orders.isEmpty(); }

    // 撮合時更新總數量
    public void decreaseQty(int qty) {
        totalQty -= qty;
    }
}
```

## 五、完整 OrderBook：支援所有訂單類型

```java
// OrderBook.java
import java.util.*;

public class OrderBook {
    private final String symbol;

    // 買單：價格高的在前（Comparator.reverseOrder）
    private final TreeMap<Double, PriceLevel> bids =
        new TreeMap<>(Comparator.reverseOrder());

    // 賣單：價格低的在前（自然順序）
    private final TreeMap<Double, PriceLevel> asks = new TreeMap<>();

    // 訂單查找表（支援快速取消）
    private final Map<String, Order> orderMap = new HashMap<>();

    private final List<Trade> tradeHistory = new ArrayList<>();

    public OrderBook(String symbol) {
        this.symbol = symbol;
    }

    // ────────────────── 主入口 ──────────────────

    public List<Trade> addOrder(Order order) {
        order.setStatus(OrderStatus.ACTIVE);
        List<Trade> trades = new ArrayList<>();

        switch (order.getType()) {
            case LIMIT  -> trades.addAll(processLimitOrder(order));
            case MARKET -> trades.addAll(processMarketOrder(order));
            case IOC    -> trades.addAll(processIOCOrder(order));
            case FOK    -> trades.addAll(processFOKOrder(order));
        }

        tradeHistory.addAll(trades);
        return trades;
    }

    // ────────────────── 限價單 ──────────────────

    private List<Trade> processLimitOrder(Order order) {
        List<Trade> trades = matchOrder(order);

        // 若有剩餘數量，加入訂單簿等待
        if (order.getRemainingQty() > 0
                && order.getStatus() != OrderStatus.FILLED) {
            addToBook(order);
        }
        return trades;
    }

    // ────────────────── 市價單 ──────────────────

    private List<Trade> processMarketOrder(Order order) {
        // 市價單不設限價，吃掉對面所有可用數量
        List<Trade> trades = matchOrder(order);

        // 市價單不進訂單簿，剩餘數量直接取消
        if (order.getRemainingQty() > 0) {
            order.setStatus(OrderStatus.CANCELLED);
        }
        return trades;
    }

    // ────────────────── IOC（立即成交或取消）──────────────────

    private List<Trade> processIOCOrder(Order order) {
        // 和限價單一樣先嘗試撮合
        List<Trade> trades = matchOrder(order);

        // 不管是否全部成交，剩餘部分一律取消（不進訂單簿）
        if (order.getRemainingQty() > 0) {
            order.setStatus(OrderStatus.CANCELLED);
        }
        return trades;
    }

    // ────────────────── FOK（全部成交或取消）──────────────────

    private List<Trade> processFOKOrder(Order order) {
        // 先檢查：對面的訂單簿是否有足夠數量可以全部成交？
        TreeMap<Double, PriceLevel> opposite = order.getSide().equals("BUY") ? asks : bids;
        int availableQty = calculateAvailableQty(order, opposite);

        if (availableQty < order.getRemainingQty()) {
            // 無法全部成交，整筆取消
            order.setStatus(OrderStatus.CANCELLED);
            return Collections.emptyList();
        }

        // 可以全部成交，執行撮合
        return matchOrder(order);
    }

    // ────────────────── 計算可撮合數量（FOK 使用）──────────────────

    private int calculateAvailableQty(Order order,
                                       TreeMap<Double, PriceLevel> opposite) {
        int available = 0;
        for (Map.Entry<Double, PriceLevel> entry : opposite.entrySet()) {
            double levelPrice = entry.getKey();

            // 限價 FOK：只計算在限價範圍內的數量
            if (order.getType() == OrderType.FOK && order.getPrice() > 0) {
                if (order.getSide().equals("BUY") && levelPrice > order.getPrice()) break;
                if (order.getSide().equals("SELL") && levelPrice < order.getPrice()) break;
            }

            available += entry.getValue().getTotalQty();
            if (available >= order.getRemainingQty()) break;
        }
        return available;
    }

    // ────────────────── 核心撮合邏輯 ──────────────────

    private List<Trade> matchOrder(Order incomingOrder) {
        List<Trade> trades = new ArrayList<>();
        TreeMap<Double, PriceLevel> opposite =
            incomingOrder.getSide().equals("BUY") ? asks : bids;

        while (incomingOrder.getRemainingQty() > 0 && !opposite.isEmpty()) {
            Map.Entry<Double, PriceLevel> bestEntry = opposite.firstEntry();
            double bestPrice = bestEntry.getKey();
            PriceLevel level = bestEntry.getValue();

            // 價格匹配檢查（市價單跳過此檢查）
            if (incomingOrder.getType() == OrderType.LIMIT
                    || incomingOrder.getType() == OrderType.IOC
                    || incomingOrder.getType() == OrderType.FOK) {
                if (incomingOrder.getSide().equals("BUY")
                        && bestPrice > incomingOrder.getPrice()) break;
                if (incomingOrder.getSide().equals("SELL")
                        && bestPrice < incomingOrder.getPrice()) break;
            }

            // 時間優先：取同一價格中最早的訂單
            Queue<Order> ordersAtLevel = level.getOrders();
            while (!ordersAtLevel.isEmpty()
                    && incomingOrder.getRemainingQty() > 0) {
                Order restingOrder = ordersAtLevel.peek();
                int matchQty = Math.min(
                    incomingOrder.getRemainingQty(),
                    restingOrder.getRemainingQty()
                );

                // 成交價格：以 resting order 的價格為準（價格優先原則）
                Trade trade;
                if (incomingOrder.getSide().equals("BUY")) {
                    trade = new Trade(
                        incomingOrder.getOrderId(),
                        restingOrder.getOrderId(),
                        bestPrice, matchQty
                    );
                } else {
                    trade = new Trade(
                        restingOrder.getOrderId(),
                        incomingOrder.getOrderId(),
                        bestPrice, matchQty
                    );
                }
                trades.add(trade);

                // 更新數量
                incomingOrder.fill(matchQty);
                restingOrder.fill(matchQty);
                level.decreaseQty(matchQty);

                // 若 resting order 全部成交，移出佇列
                if (restingOrder.getStatus() == OrderStatus.FILLED) {
                    ordersAtLevel.poll();
                    orderMap.remove(restingOrder.getOrderId());
                }
            }

            // 若此價格層已清空，移出訂單簿
            if (level.isEmpty()) {
                opposite.remove(bestPrice);
            }
        }

        return trades;
    }

    // ────────────────── 加入訂單簿 ──────────────────

    private void addToBook(Order order) {
        TreeMap<Double, PriceLevel> book =
            order.getSide().equals("BUY") ? bids : asks;

        book.computeIfAbsent(order.getPrice(), PriceLevel::new)
            .addOrder(order);
        orderMap.put(order.getOrderId(), order);
    }

    // ────────────────── 取消訂單 ──────────────────

    public boolean cancelOrder(String orderId) {
        Order order = orderMap.get(orderId);
        if (order == null
                || order.getStatus() == OrderStatus.FILLED
                || order.getStatus() == OrderStatus.CANCELLED) {
            return false;  // 訂單不存在或已不可取消
        }

        // 從訂單簿中移除
        TreeMap<Double, PriceLevel> book =
            order.getSide().equals("BUY") ? bids : asks;
        PriceLevel level = book.get(order.getPrice());
        if (level != null) {
            level.removeOrder(order);
            if (level.isEmpty()) {
                book.remove(order.getPrice());
            }
        }

        order.setStatus(OrderStatus.CANCELLED);
        orderMap.remove(orderId);
        return true;
    }

    // ────────────────── 查詢方法 ──────────────────

    // 最優買價
    public double getBestBid() {
        return bids.isEmpty() ? 0.0 : bids.firstKey();
    }

    // 最優賣價
    public double getBestAsk() {
        return asks.isEmpty() ? 0.0 : asks.firstKey();
    }

    // 買賣價差
    public double getSpread() {
        if (bids.isEmpty() || asks.isEmpty()) return -1;
        return getBestAsk() - getBestBid();
    }

    // 印出訂單簿（Level 2 行情）
    public void printBook() {
        System.out.println("═══ " + symbol + " 訂單簿 ═══");
        System.out.println("--- 賣單（Asks）---");
        asks.entrySet().stream()
            .sorted(Map.Entry.<Double, PriceLevel>comparingByKey().reversed())
            .forEach(e -> System.out.printf(
                "  %.2f | %d 股%n", e.getKey(), e.getValue().getTotalQty()));
        System.out.printf("    ── 價差 %.2f ──%n", getSpread());
        System.out.println("--- 買單（Bids）---");
        bids.forEach((price, level) -> System.out.printf(
            "  %.2f | %d 股%n", price, level.getTotalQty()));
    }

    public List<Trade> getTradeHistory() { return tradeHistory; }
}
```

## 八、加入多執行緒安全（LMAX Disruptor 模式）

```java
// 為何不用 synchronized？
// → 鎖競爭在高并發下會讓吞吐量崩潰

// 解法：單一消費者執行緒模式
// 所有訂單先進入無鎖佇列，由單一撮合執行緒依序處理

import java.util.concurrent.*;

public class MatchingEngineRunner {
    // 無界無鎖佇列（生產者安全並發寫入，消費者單執行緒讀取）
    private final BlockingQueue<Order> incomingOrders =
        new LinkedBlockingQueue<>(100_000);

    private final OrderBook orderBook = new OrderBook("TSLA");

    // 撮合執行緒（單一，不需要鎖）
    private final Thread matchingThread = new Thread(() -> {
        while (!Thread.currentThread().isInterrupted()) {
            try {
                Order order = incomingOrders.take();  // 阻塞直到有訂單
                List<Trade> trades = orderBook.addOrder(order);
                for (Trade t : trades) {
                    publishTrade(t);  // 廣播成交結果
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    });

    public void start() {
        matchingThread.setDaemon(false);
        matchingThread.start();
    }

    // 多個執行緒可以安全地呼叫這個方法（生產者）
    public void submitOrder(Order order) throws InterruptedException {
        incomingOrders.put(order);  // 佇列滿時阻塞
    }

    private void publishTrade(Trade trade) {
        System.out.println("成交：" + trade);
        // 實際系統：發送到 UDP Multicast、FIX 回報通道等
    }
}
```

```
架構示意：

多個客戶端執行緒
  ↓ ↓ ↓
  submitOrder()（執行緒安全）
  ↓
LinkedBlockingQueue（無鎖佇列）
  ↓
matchingThread（單一執行緒，不需要鎖）
  ↓
OrderBook.addOrder()（單執行緒存取，零鎖競爭）
  ↓
publishTrade()（廣播成交）
```

## 本章練習題

**Q1：FOK 訂單和 IOC 訂單的差異是什麼？各自適合什麼場景？**
<details>
<summary>答案</summary>
IOC（Immediate Or Cancel）：立即嘗試撮合，成交的部分保留，未成交的部分取消。適合「想快點成交，但接受部分成交」的場景，例如大宗訂單拆單。FOK（Fill Or Kill）：必須全部成交，否則整筆取消。適合「全部成交才有意義，部分成交沒用」的場景，例如套利交易（需要同時買一個市場賣另一個市場，數量必須匹配）。
</details>

**Q2：為什麼 addToBook 中用 computeIfAbsent 而不是 get + put？**
<details>
<summary>答案</summary>
computeIfAbsent(key, mappingFunction) 是原子操作：「如果不存在就建立」。用 get + put 分兩步會有 TOCTOU 問題（雖然在單執行緒撮合模式下不影響，但 computeIfAbsent 更清晰，也更適合未來可能的並發場景）。此外 PriceLevel::new 是方法參考，更簡潔。
</details>

**Q3：這個 OrderBook 實作在什麼情況下效能最差？如何改善？**
<details>
<summary>答案</summary>
效能最差的情況：(1) 訂單簿非常深（幾千個價格層級），TreeMap 的 O(log n) 查找積累；(2) 大量取消訂單時，HashMap 和 TreeMap 都要更新；(3) 大市價單需要吃掉多個價格層級，遍歷成本高。改善方案：(1) 對固定 tick-size 的市場，可用 int 陣列索引代替 TreeMap（O(1) 查找）；(2) 取消訂單用 lazy deletion（標記取消，撮合時跳過），避免即時從 TreeMap 移除；(3) ArrayDeque 代替 LinkedList 提升 Cache 命中率。
</details>

**Q4：描述一個完整的「買單觸發撮合」流程，從 addOrder 呼叫到 Trade 產生。**
<details>
<summary>答案</summary>
1. addOrder(buyLimitOrder) 被呼叫，設定 status=ACTIVE
2. 進入 processLimitOrder，呼叫 matchOrder
3. matchOrder 取出 asks TreeMap 的 firstEntry（最低賣價）
4. 比較：買單限價 >= 賣單最低價 → 可以撮合
5. 取 PriceLevel 的 Queue.peek()（時間最早的賣單）
6. 計算成交數量 = min(買單剩餘, 賣單剩餘)
7. 建立 Trade 物件，成交價格為賣單限價
8. 呼叫 buyOrder.fill(qty) 和 sellOrder.fill(qty) 更新剩餘數量
9. 若賣單已全部成交，從 Queue 移除，從 orderMap 移除
10. 若賣單價格層已空，從 asks 移除
11. 若買單還有剩餘且沒有更多可撮合的賣單，呼叫 addToBook 加入 bids
12. 回傳 trades 列表
</details>
