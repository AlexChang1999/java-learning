## 前言：為什麼泛型比你想的重要

你已經寫過 `List<Order>`、`TreeMap<Double, PriceLevel>`——  
但當你看到 `<T extends Comparable<? super T>>` 時，是不是直接跳過？

這章讀完，你就能看懂 Spring、JUnit 原始碼的簽名，也能寫出真正可重用的工具類別。

## 一、從問題出發：沒有泛型的世界

```java
// Java 5 以前的 List（沒有泛型）
List orders = new ArrayList();
orders.add(new Order("O1", ...));
orders.add("我是一個字串");  // 編譯器不阻止你！

Order o = (Order) orders.get(0);  // 每次取出都要強制轉型
Order bad = (Order) orders.get(1); // ← ClassCastException，執行時才爆！
```

```java
// 有泛型之後
List<Order> orders = new ArrayList<>();
orders.add(new Order("O1", ...));
orders.add("我是一個字串");  // ← 編譯時期就報錯！

Order o = orders.get(0);  // 不需要轉型，編譯器幫你確保型別
```

**泛型的本質：把型別錯誤從「執行時崩潰」提前到「編譯時報錯」。**

## 二、泛型類別與泛型方法

### 泛型類別

```java
// 自訂一個泛型 Pair 類別
public class Pair<A, B> {
    private final A first;
    private final B second;

    public Pair(A first, B second) {
        this.first = first;
        this.second = second;
    }

    public A getFirst()  { return first; }
    public B getSecond() { return second; }

    @Override
    public String toString() {
        return "(" + first + ", " + second + ")";
    }
}

// 使用：型別在建立時指定
Pair<String, Integer> nameAge = new Pair<>("Alex", 25);
Pair<Double, Order>   priceOrder = new Pair<>(350.0, new Order(...));

// 編譯器知道 first 是 String，second 是 Integer
String name = nameAge.getFirst();    // 不需要轉型
int    age  = nameAge.getSecond();
```

### 泛型方法

```java
public class Utils {

    // 泛型方法：<T> 宣告在回傳型別前面
    public static <T> List<T> repeat(T item, int times) {
        List<T> result = new ArrayList<>();
        for (int i = 0; i < times; i++) {
            result.add(item);
        }
        return result;
    }

    // 交換陣列兩個元素（任意型別都能用）
    public static <T> void swap(T[] arr, int i, int j) {
        T temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}

// 使用：編譯器自動推斷 T 的型別
List<String> threeHellos = Utils.repeat("Hello", 3);
List<Order>  fiveOrders  = Utils.repeat(new Order(...), 5);

String[] arr = {"a", "b", "c"};
Utils.swap(arr, 0, 2);  // ["c", "b", "a"]
```

## 三、型別邊界（Bounded Type Parameters）

### Upper Bound：`<T extends SomeClass>`

```java
// 只接受「可比較的」型別（實作 Comparable 的型別）
public static <T extends Comparable<T>> T max(T a, T b) {
    return a.compareTo(b) >= 0 ? a : b;
}

// 使用
String s = max("apple", "banana");  // "banana"
int    n = max(10, 20);              // 20
Order  o = max(order1, order2);      // 前提：Order 實作 Comparable<Order>

// 編譯器阻止你傳入無法比較的型別：
// max(new Object(), new Object());  ← 編譯錯誤！Object 沒有 implements Comparable
```

```java
// 在撮合引擎中的應用：找最佳價格
public static <T extends Comparable<T>> T findBest(List<T> items) {
    if (items.isEmpty()) throw new NoSuchElementException();
    T best = items.get(0);
    for (T item : items) {
        if (item.compareTo(best) > 0) best = item;
    }
    return best;
}
```

### Multiple Bounds：同時限制多個條件

```java
// T 必須同時繼承 Serializable 且實作 Comparable
public <T extends Serializable & Comparable<T>> void saveAndSort(List<T> items) {
    Collections.sort(items);
    // ... 序列化到磁碟
}
// 注意：類別放最前面，介面放後面，以 & 分隔
```

## 四、通配符（Wildcards）：最讓人頭痛的部分

### 問題：為什麼 `List<Dog>` 不是 `List<Animal>`？

```java
// 現實生活中：狗是動物（Dog extends Animal），所以你會以為：
List<Dog> dogs = new ArrayList<>();
List<Animal> animals = dogs;  // ← 編譯錯誤！為什麼？

// 因為如果這樣允許的話：
animals.add(new Cat());  // 合法！Cat 也是 Animal
Dog d = dogs.get(0);     // 取出來的其實是 Cat，ClassCastException！
// 泛型是不協變的（invariant），保護你不犯這個錯
```

### `? extends T`（上界通配符）：「只讀」情境

```java
// 計算任意數字列表的總和
// 用 ? extends Number 表示：「接受 List<Integer>、List<Double>、List<Long> 等」
public static double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}

// 使用
sum(List.of(1, 2, 3));          // List<Integer> ✅
sum(List.of(1.0, 2.5, 3.7));    // List<Double>  ✅
sum(List.of(100L, 200L));        // List<Long>    ✅

// 為什麼不能寫入？
List<? extends Number> nums = new ArrayList<Integer>();
nums.add(3.14);  // ← 編譯錯誤！編譯器不知道 List 裡實際存什麼型別
// 生產者 extends（PECS 原則的前半段）
```

### `? super T`（下界通配符）：「只寫」情境

```java
// 把 List<Integer> 的元素複製到目標列表
// 目標可以是 List<Integer>、List<Number>、List<Object>
public static void copyIntegers(List<Integer> src, List<? super Integer> dest) {
    for (Integer i : src) {
        dest.add(i);   // 寫入 Integer 一定沒問題（因為 dest 是 Integer 或更寬泛的型別）
    }
}

List<Number>  numbers = new ArrayList<>();
List<Object>  objects = new ArrayList<>();
List<Integer> ints    = List.of(1, 2, 3);

copyIntegers(ints, numbers);  // ✅ List<Number> super Integer
copyIntegers(ints, objects);  // ✅ List<Object> super Integer
// 消費者 super（PECS 原則的後半段）
```

### PECS 原則（Producer Extends, Consumer Super）

```
記憶口訣：
  如果你要「讀」資料（它是生產者）→ 用 ? extends T
  如果你要「寫」資料（它是消費者）→ 用 ? super T
  如果既讀又寫                     → 用具體型別 T

Collections.copy() 的簽名就是 PECS 的完美範例：
public static <T> void copy(List<? super T> dest,   // 消費者（寫入）
                             List<? extends T> src)  // 生產者（讀取）
```

## 五、型別擦除（Type Erasure）：泛型的秘密

```
Java 泛型在「編譯後」會被擦除！

你寫的：
  List<Order> orders = new ArrayList<Order>();
  orders.add(order);
  Order o = orders.get(0);

編譯後的 bytecode 實際上是：
  List orders = new ArrayList();       // 型別參數消失了
  orders.add(order);
  Order o = (Order) orders.get(0);     // 編譯器自動插入轉型
```

**型別擦除的後果：**

```java
// 1. 執行時無法知道泛型的實際型別
List<Order> orders = new ArrayList<>();
System.out.println(orders.getClass());  // class java.util.ArrayList（沒有 <Order>）

// 2. 不能用泛型型別建立物件
public <T> T createInstance() {
    return new T();  // ← 編譯錯誤！擦除後 T 是 Object，不知道要 new 什麼
}

// 解法：傳入 Class<T>
public <T> T createInstance(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();  // 用反射（第十八章）
}
Order o = createInstance(Order.class);

// 3. 不能用 instanceof 檢查泛型型別
if (obj instanceof List<Order>) { }  // ← 編譯錯誤！
if (obj instanceof List<?>)     { }  // ← 這樣才對（用無界通配符）
```

## 六、泛型與集合框架的完整關係

```java
// 你常用的這些，底層都是泛型：

// TreeMap 的完整簽名：
TreeMap<Double, PriceLevel> bids = new TreeMap<>(Comparator.reverseOrder());
// K=Double, V=PriceLevel

// Comparator 是函數式介面（泛型）：
Comparator<Order> byPrice = (a, b) -> Double.compare(a.getPrice(), b.getPrice());

// Optional 是泛型容器：
Optional<Order> bestBid = Optional.ofNullable(book.getBestBidOrder());

// Stream 的中間操作全是泛型方法：
List<Double> prices = orders.stream()
    .filter(o -> o.getSide().equals("BUY"))    // Stream<Order> → Stream<Order>
    .map(Order::getPrice)                        // Stream<Order> → Stream<Double>
    .collect(Collectors.toList());               // Stream<Double> → List<Double>
```

## 本章練習題

**Q1：以下程式碼能編譯嗎？為什麼？**
```java
List<Integer> ints = new ArrayList<>();
List<Number> nums = ints;
```
<details>
<summary>答案</summary>
不能編譯。泛型是不協變的（invariant），即使 Integer extends Number，List&lt;Integer&gt; 也不是 List&lt;Number&gt; 的子型別。這是為了型別安全：如果允許這個賦值，你就能透過 nums.add(3.14) 把 Double 放進 List&lt;Integer&gt;，造成執行時 ClassCastException。正確做法是使用 List&lt;? extends Number&gt; nums = ints;（只讀）。
</details>

**Q2：`<T extends Comparable<T>>` 和 `<T extends Comparable<? super T>>` 有什麼差異？**
<details>
<summary>答案</summary>
前者要求 T 和 Comparable&lt;T&gt; 完全匹配。後者更寬鬆：T 只需要能和「T 的某個父類別」比較。例如：Apple extends Fruit，Fruit implements Comparable&lt;Fruit&gt;——Apple 沒有直接實作 Comparable&lt;Apple&gt;，用前者會編譯失敗，用後者可以。Java 標準庫的 Collections.sort() 就用後者，讓所有繼承鏈都能正確運作。
</details>

**Q3：為什麼不能寫 `new T()`？要怎麼繞過這個限制？**
<details>
<summary>答案</summary>
因為型別擦除：執行時 T 變成 Object，JVM 不知道該 new 什麼。繞過的方法是傳入 Class&lt;T&gt; clazz 參數，再用 clazz.getDeclaredConstructor().newInstance() 透過反射建立。這也是 Spring 內部大量使用的模式——它透過 @Component 掃描 + Class 物件來建立 Bean 實例。
</details>
