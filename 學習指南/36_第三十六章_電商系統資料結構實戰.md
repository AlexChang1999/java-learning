## 一、電商系統的特徵

```
電商系統的流量特性：
  平時：讀多寫少（每 1 筆下單，可能有 100 次商品瀏覽）
  活動（雙11/週年慶）：瞬間超高並發（秒殺、搶券）
  資料規模：商品 TB 級，用戶億級，訂單數十億

這些特性決定了：
  ✅ 讀路徑：需要 Cache（Redis），不能每次打 DB
  ✅ 寫路徑：需要高並發（樂觀鎖、隊列削峰）
  ✅ 搜尋：倒排索引（Elasticsearch 底層）
  ✅ 推薦：圖資料結構（協同過濾）
```

## 二、商品搜尋 — 倒排索引（Inverted Index） <!-- 💡 進階 -->

### 正向索引 vs 倒排索引

```
正向索引（Forward Index）：「文件 → 包含哪些詞」
  文件1："iPhone 手機 蘋果"
  文件2："手機 三星 Android"
  文件3："蘋果 MacBook 筆電"

  查詢「蘋果」：需要掃描所有文件 → O(n)，太慢！

倒排索引（Inverted Index）：「詞 → 包含這個詞的哪些文件」
  "iPhone"  → [文件1]
  "手機"    → [文件1, 文件2]
  "蘋果"    → [文件1, 文件3]  ← 查詢 O(1)！
  "三星"    → [文件2]
  "MacBook" → [文件3]

查詢「蘋果手機」：
  "蘋果" 的文件集合 ∩ "手機" 的文件集合
  = {文件1, 文件3} ∩ {文件1, 文件2}
  = {文件1}  ← 只有文件1 同時包含「蘋果」和「手機」
```

### Java 簡易倒排索引實作

```java
public class InvertedIndex {

    // 詞 → 包含這個詞的商品ID列表（用 TreeSet 保持有序，方便做排名）
    private final Map<String, TreeSet<Long>> index = new HashMap<>();

    // 商品ID → 商品資訊
    private final Map<Long, Product> products = new HashMap<>();

    /** 建立索引：對商品名稱做分詞後建立倒排 */
    public void indexProduct(Product product) {
        products.put(product.getId(), product);

        // 分詞（生產環境用 Lucene / IK Analyzer 做中文分詞）
        String[] tokens = tokenize(product.getName());

        for (String token : tokens) {
            index.computeIfAbsent(token, k -> new TreeSet<>())
                 .add(product.getId());
        }
    }

    /** 搜尋：支援多關鍵字（AND 邏輯） */
    public List<Product> search(String query) {
        String[] tokens = tokenize(query);
        if (tokens.length == 0) return Collections.emptyList();

        // 從第一個關鍵字的結果開始
        Set<Long> result = new HashSet<>(
            index.getOrDefault(tokens[0], new TreeSet<>()));

        // 與後續關鍵字的結果做交集（AND）
        for (int i = 1; i < tokens.length; i++) {
            Set<Long> next = index.getOrDefault(tokens[i], new TreeSet<>());
            result.retainAll(next);   // 取交集
            if (result.isEmpty()) return Collections.emptyList();  // 提早返回
        }

        return result.stream()
            .map(products::get)
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
    }

    private String[] tokenize(String text) {
        // 簡化版：按空格分詞
        // 生產環境：IK Analyzer、jieba 等中文分詞器
        return text.toLowerCase().split("\\s+");
    }
}
```

### Elasticsearch 的底層就是倒排索引

```
用戶搜尋「Nike 跑鞋 40碼」：

Elasticsearch 倒排索引：
  "nike"    → [商品#1023, 商品#1045, ...]
  "跑鞋"   → [商品#1023, 商品#2001, ...]
  "40碼"   → [商品#1023, 商品#3005, ...]
  交集      → [商品#1023]

再根據 TF-IDF 或 BM25 算相關度評分，排序後返回

硬體視角：
  倒排索引文件存在 SSD（~100μs / 次 IO）
  熱門關鍵字的 Posting List 在 OS Page Cache（~100ns）
  → 常見搜尋延遲 < 10ms
```

## 三、購物車 — HashMap + Redis <!-- 入門 -->

### 購物車的資料結構

```
需求：
  ✅ 加入商品：O(1)
  ✅ 移除商品：O(1)
  ✅ 查看數量：O(1)
  ✅ 計算總價：O(n)，n = 購物車中商品種類數

資料結構：HashMap<商品ID, CartItem>（商品ID → 購物車項目）
```

```java
@Service
public class CartService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    private String cartKey(String userId) {
        return "cart:" + userId;
    }

    /** 加入商品（或增加數量），O(1) */
    public void addItem(String userId, String productId, int quantity) {
        String key = cartKey(userId);
        // Redis Hash：HGET cart:user123 productId
        Object existing = redisTemplate.opsForHash().get(key, productId);

        int newQty = quantity;
        if (existing != null) {
            newQty += (int) existing;   // 累加數量
        }
        redisTemplate.opsForHash().put(key, productId, newQty);

        // 設定購物車 30 天過期（未結帳就清空）
        redisTemplate.expire(key, Duration.ofDays(30));
    }

    /** 取得購物車所有商品，O(n) */
    public Map<String, Integer> getCart(String userId) {
        Map<Object, Object> entries =
            redisTemplate.opsForHash().entries(cartKey(userId));

        return entries.entrySet().stream()
            .collect(Collectors.toMap(
                e -> (String) e.getKey(),
                e -> (Integer) e.getValue()
            ));
    }

    /** 計算購物車總價，O(n) */
    public BigDecimal calculateTotal(String userId) {
        Map<String, Integer> cart = getCart(userId);

        // 批次查詢商品價格（Pipeline 減少 Redis RTT）
        return cart.entrySet().stream()
            .map(e -> {
                Product p = productService.getById(e.getKey());
                return p.getPrice().multiply(BigDecimal.valueOf(e.getValue()));
            })
            .reduce(BigDecimal.ZERO, BigDecimal::add);
    }
}
```

## 四、排行榜 — Redis ZSet（跳表） <!-- 💡 進階 -->

### 場景：即時銷量排行榜

```
需求：
  ✅ 更新某商品銷量：O(log n)
  ✅ 取前 10 名：O(log n + 10)
  ✅ 查詢某商品排名：O(log n)
  ✅ 並發更新安全
  
為什麼用 Redis ZSet 而不是 Java TreeMap？
  TreeMap 是單機的，排行榜需要多台伺服器都看到同樣的資料
  ZSet 底層是跳表，支援並發讀取
```

```java
@Service
public class SalesRankingService {

    private static final String SALES_RANK_KEY = "ranking:sales:daily";

    @Autowired
    private StringRedisTemplate redisTemplate;

    /** 商品賣出後，更新銷量，O(log n) */
    public void recordSale(String productId, long quantity) {
        // ZINCRBY key increment member
        // ZSet 的 score = 銷量，member = 商品ID
        redisTemplate.opsForZSet().incrementScore(SALES_RANK_KEY, productId, quantity);
    }

    /** 取銷量前 N 名，O(log n + N) */
    public List<RankItem> getTopN(int n) {
        // ZREVRANGEBYSCORE：按 score 降序取前 n 名，含 score
        Set<ZSetOperations.TypedTuple<String>> topItems =
            redisTemplate.opsForZSet()
                .reverseRangeWithScores(SALES_RANK_KEY, 0, n - 1);

        List<RankItem> result = new ArrayList<>();
        int rank = 1;
        for (ZSetOperations.TypedTuple<String> item : topItems) {
            result.add(new RankItem(rank++, item.getValue(),
                                    item.getScore().longValue()));
        }
        return result;
    }

    /** 查詢某商品的排名，O(log n) */
    public Long getRank(String productId) {
        Long rank = redisTemplate.opsForZSet()
            .reverseRank(SALES_RANK_KEY, productId);
        return rank == null ? -1 : rank + 1;  // ZSet 從 0 開始，+1 轉成人類可讀的排名
    }

    /** 每日凌晨重置，建立昨日 Key 保留歷史 */
    @Scheduled(cron = "0 0 0 * * *")
    public void dailyReset() {
        String yesterday = LocalDate.now().minusDays(1).toString();
        redisTemplate.rename(SALES_RANK_KEY, "ranking:sales:" + yesterday);
        redisTemplate.expire("ranking:sales:" + yesterday, Duration.ofDays(30));
        // 新的 Key 自動建立（下次 ZINCRBY 就會建）
    }
}
```

## 五、秒殺系統 — 高並發下的庫存扣減 <!-- 🔴 資深 -->

### 問題：100 個人同時搶 1 件商品

```
Race Condition（競態條件）：

Thread-A: SELECT stock FROM items WHERE id=1;  → 得到 1
Thread-B: SELECT stock FROM items WHERE id=1;  → 得到 1
Thread-A: UPDATE items SET stock=0 WHERE id=1;
Thread-B: UPDATE items SET stock=0 WHERE id=1;  ← 超賣！兩人都成功

解法有三：
  1. 悲觀鎖（SELECT FOR UPDATE）：排隊，吞吐量低
  2. 樂觀鎖（CAS）：用版本號，失敗重試
  3. Redis 原子操作：最快
```

### 解法一：樂觀鎖（CAS）

```java
// 數據庫表：items(id, stock, version)

@Transactional
public boolean deductStock(Long itemId) {
    Item item = itemMapper.selectById(itemId);
    if (item.getStock() <= 0) return false;

    // UPDATE items SET stock = stock - 1, version = version + 1
    // WHERE id = ? AND version = ? AND stock > 0
    int affected = itemMapper.updateByOptimisticLock(
        itemId, item.getVersion());  // 只有 version 匹配才更新

    if (affected == 0) {
        // 版本衝突（被別人搶先了），重試
        throw new OptimisticLockException("庫存競爭，請重試");
    }
    return true;
}
```

### 解法二：Redis Lua Script（最推薦） <!-- 🔴 資深 -->

```java
/**
 * 原子扣減庫存：Lua Script 在 Redis 單線程執行，不會有並發問題
 * 比 SELECT FOR UPDATE 快 10-100 倍（記憶體操作 vs 磁碟鎖）
 */
@Service
public class SeckillService {

    // 預先載入庫存到 Redis
    public void initStock(Long itemId, int stock) {
        redisTemplate.opsForValue().set("stock:" + itemId, String.valueOf(stock));
    }

    private static final String DEDUCT_STOCK_LUA = """
        local stock = tonumber(redis.call('get', KEYS[1]))
        if stock == nil then return -1 end     -- key 不存在
        if stock <= 0 then return 0 end        -- 庫存不足
        redis.call('decr', KEYS[1])            -- 扣減庫存
        return 1                               -- 成功
        """;

    public SeckillResult seckill(Long userId, Long itemId) {
        // 1. 原子扣減 Redis 庫存（ns 級，幾乎不會失敗）
        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(DEDUCT_STOCK_LUA, Long.class),
            List.of("stock:" + itemId)
        );

        if (result == null || result <= 0) {
            return SeckillResult.SOLD_OUT;   // 已售罄
        }

        // 2. 發送訊息到 Kafka，非同步建立訂單（不阻塞秒殺請求）
        kafkaTemplate.send("seckill-orders",
            new SeckillEvent(userId, itemId, System.currentTimeMillis()));

        return SeckillResult.SUCCESS;
    }
}
```

### 秒殺架構圖

```
用戶請求 → Nginx → Gateway（限流：令牌桶）→ Seckill Service
                                                    │
                                                    ↓
                                              Redis（庫存）
                                              Lua 原子扣減
                                                    │
                                          ┌─────────┴──────────┐
                                       成功                   失敗
                                          │                    │
                                    Kafka（非同步）          返回「已售罄」
                                          │
                                    Order Service
                                    （建立訂單 + 扣 DB 庫存）

關鍵點：
  Redis 承受秒殺洪峰（記憶體，10萬 QPS 輕鬆）
  DB 只收到 Kafka 的有序訊息（不被洪峰打）
  令牌桶限流：保護後端，多餘請求直接拒絕
```

## 六、推薦系統 — 圖與矩陣 <!-- 🔴 資深 -->

### 協同過濾（Collaborative Filtering）

```
思路：「和你買了相似商品的人，還買了什麼」

用戶-商品矩陣：
           商品A  商品B  商品C  商品D
  用戶1  [  5,    4,    0,    1  ]   （0 = 未購買）
  用戶2  [  4,    0,    4,    1  ]
  用戶3  [  1,    1,    5,    0  ]

找和「用戶1」最相似的用戶（向量餘弦相似度）：
  用戶1 vs 用戶2：cos(用戶1, 用戶2) = 0.87  ← 相似！
  用戶1 vs 用戶3：cos(用戶1, 用戶3) = 0.21

推薦：用戶2 買了商品C，用戶1 還沒買 → 推薦商品C 給用戶1
```

```java
public class CollaborativeFilter {

    // 用 HashMap 存稀疏矩陣（大多數用戶只買少數商品）
    // userId → (productId → rating)
    private final Map<Long, Map<Long, Double>> userItemMatrix = new HashMap<>();

    public void record(Long userId, Long productId, double rating) {
        userItemMatrix.computeIfAbsent(userId, k -> new HashMap<>())
                      .put(productId, rating);
    }

    /** 餘弦相似度：值域 [-1, 1]，越接近 1 越相似 */
    public double cosineSimilarity(Long userA, Long userB) {
        Map<Long, Double> vecA = userItemMatrix.getOrDefault(userA, Map.of());
        Map<Long, Double> vecB = userItemMatrix.getOrDefault(userB, Map.of());

        // 只計算共同評分的商品（稀疏矩陣優化）
        double dotProduct = 0, normA = 0, normB = 0;
        for (Map.Entry<Long, Double> e : vecA.entrySet()) {
            normA += e.getValue() * e.getValue();
            if (vecB.containsKey(e.getKey())) {
                dotProduct += e.getValue() * vecB.get(e.getKey());
            }
        }
        for (double v : vecB.values()) normB += v * v;

        if (normA == 0 || normB == 0) return 0;
        return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
    }

    /** 找 Top-K 相似用戶 */
    public List<Long> findSimilarUsers(Long userId, int k) {
        PriorityQueue<long[]> minHeap =   // [similarity * 1000, userId]（最小堆）
            new PriorityQueue<>(Comparator.comparingLong(a -> a[0]));

        for (Long otherUser : userItemMatrix.keySet()) {
            if (otherUser.equals(userId)) continue;
            double sim = cosineSimilarity(userId, otherUser);
            minHeap.offer(new long[]{(long)(sim * 1000), otherUser});
            if (minHeap.size() > k) minHeap.poll();  // 超過 k 個就丟最小的
        }

        return minHeap.stream().map(a -> a[1]).collect(Collectors.toList());
    }
}
```

## 七、電商資料結構選型速查

| 功能 | 資料結構 | Java / Redis 實作 |
|------|----------|------------------|
| 商品搜尋 | 倒排索引 | Elasticsearch / Lucene |
| 購物車 | HashMap | Redis Hash |
| 銷量排行榜 | 跳表（有序集合） | Redis ZSet |
| 秒殺庫存扣減 | Redis 原子計數器 + Lua | `DECR` + Lua Script |
| 商品分類樹 | N 叉樹 | 遞迴或 `LEFT/RIGHT` 嵌套集合 |
| 推薦系統 | 稀疏矩陣 + 相似度 | HashMap + 餘弦相似度 |
| 訂單狀態機 | 有向圖（DAG） | Enum + Map<State, List<State>> |
| 優惠券去重 | Bloom Filter | Guava BloomFilter |
