## 一、遊戲系統的特殊需求

```
遊戲系統 vs 一般業務系統的核心差異：

一般業務系統：
  ✅ 資料正確性優先
  ✅ 延遲 100ms 以內即可
  ✅ 資料持久化

遊戲系統額外要求：
  ✅ 幀同步（每幀 ~16ms，60 FPS）→ 極低延遲
  ✅ 空間查詢（「我周圍 100 單位內有哪些敵人？」）
  ✅ 確定性亂數（可重放、可稽核，對應 RTP 監管要求）
  ✅ 事件驅動（大量遊戲物件同時更新）
```

## 二、碰撞偵測 — 四叉樹（Quadtree） <!-- 💡 進階 -->

### 問題：暴力碰撞偵測太慢

```
捕魚機場景：
  場景中有 200 條魚、50 顆砲彈同時移動
  每幀需要檢查是否有碰撞

暴力做法：每顆砲彈對每條魚做距離計算
  50 × 200 = 10,000 次距離計算 / 幀
  60 FPS → 600,000 次 / 秒
  距離計算：sqrt((x1-x2)² + (y1-y2)²) → 浮點運算 → 慢！

優化問題：大多數砲彈和魚根本不在同一區域，不需要計算
```

### 四叉樹（Quadtree）

```
思路：把遊戲場景遞迴劃分成四個象限，只對「同一區塊」的物件做碰撞檢測

場景（800×600）
┌──────────────────┬──────────────────┐
│  第一象限（NW）   │  第二象限（NE）  │
│  [魚A, 魚C]      │  [魚B, 砲彈1]    │
├──────────────────┼──────────────────┤
│  第三象限（SW）   │  第四象限（SE）  │
│  [魚D, 魚E, 魚F] │  []              │
└──────────────────┴──────────────────┘

砲彈1 在 NE → 只和 [魚B] 做碰撞計算（1次，不是 200次！）
魚D/E/F 在 SW → 和砲彈1 完全不需要計算

複雜度：從 O(N×M) 降到 O((N+M) × log(N+M))
```

### Java 實作

```java
public class Quadtree {
    private static final int MAX_OBJECTS = 10;  // 每個節點最多存 10 個物件
    private static final int MAX_LEVELS  = 5;   // 最多分 5 層（防無限細分）

    private final int level;          // 當前節點的層次
    private final Rectangle bounds;  // 這個節點代表的矩形區域
    private final List<GameObject> objects = new ArrayList<>();
    private Quadtree[] children;     // 4 個子節點（null 表示未細分）

    public Quadtree(int level, Rectangle bounds) {
        this.level  = level;
        this.bounds = bounds;
    }

    /** 清空（每幀重建更簡單可靠） */
    public void clear() {
        objects.clear();
        if (children != null) {
            for (Quadtree child : children) child.clear();
            children = null;
        }
    }

    /** 細分成 4 個子象限 */
    private void split() {
        int subW = bounds.width  / 2;
        int subH = bounds.height / 2;
        int x = bounds.x, y = bounds.y;

        children = new Quadtree[4];
        children[0] = new Quadtree(level+1, new Rectangle(x + subW, y,        subW, subH)); // NE
        children[1] = new Quadtree(level+1, new Rectangle(x,        y,        subW, subH)); // NW
        children[2] = new Quadtree(level+1, new Rectangle(x,        y + subH, subW, subH)); // SW
        children[3] = new Quadtree(level+1, new Rectangle(x + subW, y + subH, subW, subH)); // SE
    }

    /** 判斷物件應該在哪個象限（-1 = 跨象限，放在父節點） */
    private int getIndex(GameObject obj) {
        int midX = bounds.x + bounds.width  / 2;
        int midY = bounds.y + bounds.height / 2;

        boolean top    = obj.y < midY && obj.y + obj.height < midY;
        boolean bottom = obj.y > midY;

        if (obj.x < midX && obj.x + obj.width < midX) {
            if (top)    return 1;  // NW
            if (bottom) return 2;  // SW
        } else if (obj.x > midX) {
            if (top)    return 0;  // NE
            if (bottom) return 3;  // SE
        }
        return -1;  // 跨象限
    }

    /** 插入物件 */
    public void insert(GameObject obj) {
        if (children != null) {
            int idx = getIndex(obj);
            if (idx != -1) {
                children[idx].insert(obj);
                return;
            }
        }

        objects.add(obj);

        // 超過容量且未達最大層數 → 細分
        if (objects.size() > MAX_OBJECTS && level < MAX_LEVELS) {
            if (children == null) split();
            // 把現有物件下移到子節點
            Iterator<GameObject> it = objects.iterator();
            while (it.hasNext()) {
                int idx = getIndex(it.next());
                if (idx != -1) {
                    children[idx].insert(it.next());
                    it.remove();
                }
            }
        }
    }

    /**
     * 取得可能與 target 碰撞的物件列表
     * 只返回「同一區域」的物件，大幅減少需要精確計算的數量
     */
    public List<GameObject> retrieve(GameObject target) {
        List<GameObject> candidates = new ArrayList<>(objects);  // 本節點的物件

        if (children != null) {
            int idx = getIndex(target);
            if (idx != -1) {
                candidates.addAll(children[idx].retrieve(target));
            } else {
                // 跨象限：查所有子節點
                for (Quadtree child : children) {
                    candidates.addAll(child.retrieve(target));
                }
            }
        }
        return candidates;
    }
}

// 每幀使用：
Quadtree qt = new Quadtree(0, new Rectangle(0, 0, 800, 600));

// 1. 重建（每幀物件都在移動，所以每幀重建）
qt.clear();
for (Fish fish : allFish) qt.insert(fish);

// 2. 碰撞檢測（每顆砲彈只和「同區域的魚」計算）
for (Bullet bullet : allBullets) {
    List<GameObject> candidates = qt.retrieve(bullet);
    for (GameObject candidate : candidates) {
        if (candidate instanceof Fish && intersects(bullet, (Fish) candidate)) {
            // 砲彈打中魚！
            handleCollision(bullet, (Fish) candidate);
        }
    }
}
```

## 三、事件系統 — Ring Buffer（環形緩衝區） <!-- 🔴 資深 -->

### 遊戲的事件驅動架構

```
遊戲中有大量事件：
  玩家按下發射鍵 → 產生 ShootEvent
  魚被擊中 → 產生 HitEvent
  魚死亡 → 產生 FishDeathEvent → 觸發金幣計算 → 觸發音效 → 觸發動畫...

如果每個事件都同步處理（做完才繼續），遊戲會卡頓

解法：Ring Buffer（環形緩衝區）
  生產者（遊戲邏輯）快速把事件放進 Buffer
  消費者（渲染、音效、網路）非同步讀取
  兩者解耦，互不阻塞
```

### Ring Buffer 原理

```
固定大小的陣列，用兩個指標（head, tail）追蹤讀寫位置：

初始：head=0, tail=0（空）
放入 E1：tail=1
         [E1][ ][ ][ ][ ][ ][ ][ ]
          ↑                    
         head=0, tail=1

放入 E2, E3：tail=3
         [E1][E2][E3][ ][ ][ ][ ][ ]
          ↑
         head=0, tail=3

消費 E1：head=1
         [  ][E2][E3][ ][ ][ ][ ][ ]
               ↑
              head=1, tail=3

放入 E4...E8（繞回）：tail % 8 = 0
         [E8][E2][E3][E4][E5][E6][E7][ ]
         tail=0  ↑
                head=1  （tail 追上 head → 滿！）

優點：
  - 固定記憶體，無 GC（陣列，不 new 物件）
  - Cache Friendly（連續記憶體）
  - 無鎖實作（LMAX Disruptor 就是這個思想）
```

### Java 簡易 Ring Buffer

```java
/**
 * 單生產者單消費者的無鎖 Ring Buffer
 * 遊戲引擎中用於事件佇列
 */
public class RingBuffer<T> {
    private final Object[] buffer;
    private final int capacity;
    private final int mask;         // capacity - 1，用於取模（要求 capacity 是 2 的冪次）

    // volatile 保證多線程可見性（不需要鎖）
    private volatile long head = 0;  // 消費者讀取位置
    private volatile long tail = 0;  // 生產者寫入位置

    public RingBuffer(int capacity) {
        // capacity 必須是 2 的冪次（方便用 & 取模，比 % 快）
        this.capacity = capacity;
        this.mask     = capacity - 1;
        this.buffer   = new Object[capacity];
    }

    /** 放入事件，O(1)，單生產者線程安全 */
    public boolean offer(T item) {
        long t = tail;
        if (t - head >= capacity) return false;  // Buffer 已滿
        buffer[(int)(t & mask)] = item;           // & mask 等效於 % capacity
        tail = t + 1;                             // volatile write，對消費者可見
        return true;
    }

    /** 取出事件，O(1)，單消費者線程安全 */
    @SuppressWarnings("unchecked")
    public T poll() {
        long h = head;
        if (h >= tail) return null;  // Buffer 空
        T item = (T) buffer[(int)(h & mask)];
        head = h + 1;
        return item;
    }

    public boolean isEmpty() { return head >= tail; }
    public int size() { return (int)(tail - head); }
}

// 使用：
RingBuffer<GameEvent> eventBuffer = new RingBuffer<>(1024);

// 遊戲邏輯線程（生產者）
eventBuffer.offer(new ShootEvent(playerId, x, y));

// 渲染線程（消費者）
GameEvent event;
while ((event = eventBuffer.poll()) != null) {
    renderSystem.process(event);
}
```

## 四、機率系統 — 確定性亂數與 RTP <!-- 🔴 資深 -->

### 什麼是 RTP（Return to Player）？

```
RTP = 理論還款率（Return to Player）
  玩家每投入 100 元，長期平均能拿回的錢
  合規要求：台灣電子遊藝場 RTP ≥ 80%（依不同地區法規）

問題：如果用 Math.random() 產生亂數，怎麼保證 RTP？
  Math.random() 是偽亂數（pseudo-random），看起來隨機，但可預測

合規要求：
  1. 亂數要可稽核（監管單位要能驗證）
  2. 長期 RTP 要在允許範圍內（不能偏差太多）
  3. 每局結果要獨立（不能「欠債」補回）
```

### 線性同餘生成器（LCG）— 最簡單的確定性亂數

```java
/**
 * 線性同餘生成器（Linear Congruential Generator）
 * 公式：X(n+1) = (a * X(n) + c) % m
 *
 * 特性：
 *   - 給定相同的 seed，輸出序列完全相同（可重放、可稽核）
 *   - 週期固定，不可能真正「隨機」
 *   - 適合非密碼學場景（遊戲足夠）
 */
public class LCGRandom {
    private static final long A = 1_664_525L;
    private static final long C = 1_013_904_223L;
    private static final long M = 1L << 32;  // 2^32

    private long seed;

    public LCGRandom(long seed) {
        this.seed = seed;
    }

    /** 產生 [0, 1) 的亂數 */
    public double nextDouble() {
        seed = (A * seed + C) % M;
        return (double) seed / M;
    }

    /** 產生 [min, max] 的整數 */
    public int nextInt(int min, int max) {
        return (int)(nextDouble() * (max - min + 1)) + min;
    }
}
```

### 加權機率抽獎（捕魚機魚種配置）

```java
/**
 * 加權隨機選取：不同魚種有不同的出現機率
 *
 * 魚種配置：
 *   普通小魚：權重 5000（出現機率 50%）
 *   中型魚：  權重 3000（30%）
 *   大魚：    權重 1500（15%）
 *   鯊魚：    權重  400（4%）
 *   龍王：    權重  100（1%）
 *   合計：    10000
 */
public class WeightedRandom {

    // 累積權重陣列（前綴和），方便用二分搜尋定位
    private final int[] cumulativeWeights;
    private final String[] fishTypes;
    private final LCGRandom rng;

    public WeightedRandom(Map<String, Integer> weightMap, long seed) {
        int n = weightMap.size();
        fishTypes = weightMap.keySet().toArray(new String[0]);
        cumulativeWeights = new int[n];
        rng = new LCGRandom(seed);

        // 建立前綴和陣列
        int[] weights = new int[n];
        for (int i = 0; i < n; i++) {
            weights[i] = weightMap.get(fishTypes[i]);
        }
        cumulativeWeights[0] = weights[0];
        for (int i = 1; i < n; i++) {
            cumulativeWeights[i] = cumulativeWeights[i-1] + weights[i];
        }
    }

    /** 抽一個魚種，O(log n)（二分搜尋） */
    public String drawFish() {
        int total  = cumulativeWeights[cumulativeWeights.length - 1];
        int target = (int)(rng.nextDouble() * total);

        // 二分搜尋：找第一個 cumulativeWeights[i] > target 的 i
        int lo = 0, hi = cumulativeWeights.length - 1;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (cumulativeWeights[mid] <= target) lo = mid + 1;
            else hi = mid;
        }
        return fishTypes[lo];
    }
}

// 使用：
Map<String, Integer> config = new LinkedHashMap<>();
config.put("普通小魚", 5000);
config.put("中型魚",   3000);
config.put("大魚",     1500);
config.put("鯊魚",      400);
config.put("龍王",      100);

WeightedRandom wr = new WeightedRandom(config, System.currentTimeMillis());

// 模擬 100,000 次，驗證 RTP
long totalBet    = 0;
long totalReturn = 0;
Map<String, Long> frequency = new HashMap<>();

for (int i = 0; i < 100_000; i++) {
    String fish = wr.drawFish();
    frequency.merge(fish, 1L, Long::sum);
    totalBet    += 10;  // 每次下注 10 元
    totalReturn += getReward(fish);  // 依魚種給獎金
}

double rtp = (double) totalReturn / totalBet;
System.out.printf("模擬 RTP: %.2f%%（理論值應在 80%%~95%%）%n", rtp * 100);
```

## 五、遊戲狀態機 — 有限狀態機（FSM） <!-- 💡 進階 -->

### 捕魚機的遊戲狀態

```
狀態轉換圖：

IDLE（閒置）
  ↓ 玩家投幣
PLAYING（遊戲中）
  ├─ 發射砲彈 → PLAYING（繼續）
  ├─ 砲彈命中魚 → FISH_HIT（魚受傷動畫）→ PLAYING
  ├─ 魚死亡 → REWARD（計算獎金）→ PLAYING
  └─ 砲彈用完 → GAME_OVER
GAME_OVER
  ├─ 再次投幣 → PLAYING
  └─ 超時 → IDLE
BONUS_ROUND（特殊關卡）
  ↓ 限時結束
PLAYING
```

```java
public enum GameState {
    IDLE, PLAYING, FISH_HIT, REWARD, GAME_OVER, BONUS_ROUND
}

public class FishingGameFSM {

    private GameState currentState = GameState.IDLE;

    // 合法的狀態轉換表：Map<當前狀態, Map<事件, 下一個狀態>>
    private static final Map<GameState, Map<String, GameState>> TRANSITIONS =
        Map.of(
            GameState.IDLE, Map.of(
                "COIN_INSERT", GameState.PLAYING
            ),
            GameState.PLAYING, Map.of(
                "BULLET_HIT_FISH", GameState.FISH_HIT,
                "BONUS_TRIGGER",   GameState.BONUS_ROUND,
                "BULLET_EMPTY",    GameState.GAME_OVER
            ),
            GameState.FISH_HIT, Map.of(
                "FISH_DEAD",      GameState.REWARD,
                "FISH_SURVIVE",   GameState.PLAYING
            ),
            GameState.REWARD, Map.of(
                "REWARD_DONE",    GameState.PLAYING
            ),
            GameState.GAME_OVER, Map.of(
                "COIN_INSERT",    GameState.PLAYING,
                "TIMEOUT",        GameState.IDLE
            ),
            GameState.BONUS_ROUND, Map.of(
                "BONUS_END",      GameState.PLAYING
            )
        );

    /** 觸發事件，轉換狀態 */
    public GameState trigger(String event) {
        Map<String, GameState> transitions = TRANSITIONS.get(currentState);
        if (transitions == null || !transitions.containsKey(event)) {
            throw new IllegalStateException(
                String.format("狀態 %s 不支援事件 %s", currentState, event));
        }
        GameState next = transitions.get(event);
        System.out.printf("[FSM] %s --[%s]--> %s%n", currentState, event, next);
        currentState = next;
        return next;
    }

    public GameState getState() { return currentState; }
}
```

## 六、遊戲資料結構選型速查

| 遊戲功能 | 資料結構 | 效能重點 |
|---------|----------|---------|
| 碰撞偵測（空間查詢） | 四叉樹（Quadtree） | O(N×M) → O((N+M) log N) |
| 事件佇列 | Ring Buffer（環形緩衝區） | O(1)，無 GC，Cache Friendly |
| 遊戲物件排序（層次/深度） | 優先佇列（Heap） | O(log n) 插入，O(1) 取最值 |
| 亂數/機率抽獎 | 前綴和 + 二分搜尋 | O(log n) 抽取，O(n) 建立 |
| 遊戲狀態控制 | 有限狀態機（FSM） | Map 查表，O(1) 轉換 |
| 場景物件管理 | HashMap<Id, GameObject> | O(1) 按 ID 存取 |
| 技能冷卻計時 | 優先佇列（依到期時間排序） | O(log n) 更新 |
| 路徑規劃（NPC 走路） | A* 算法（圖 + 優先佇列） | O((V+E) log V) |
