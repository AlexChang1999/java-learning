## 一、體育彩票的技術特徵

```
體育彩票的特殊挑戰：

即時性：
  賠率隨賽況即時變動（每秒可能更新幾十次）
  投注必須在截止時間前完成

精確性：
  賠率計算涉及大量浮點數，必須用定點數避免誤差
  獎金計算必須完全可稽核

高並發：
  重要賽事：萬人同時投注某個選項
  需要防止超賣（投注金額超過莊家能賠付的上限）

法規：
  所有投注記錄、賠率變化都必須可追溯
```

## 二、即時賠率 — Segment Tree <!-- 🔴 資深 -->

### 場景：查詢某時間段內的最高賠率

```
監管報告需求：
  「昨天下午 14:00~18:00，某場比賽的賠率最高是多少？」
  「某選項的賠率走勢（每小時的最大/最小）」

賠率是隨時間變化的序列：
  時刻0: 1.85   時刻1: 1.90   時刻2: 1.75   時刻3: 2.10   ...

區間最大/最小值查詢 + 單點更新 → Segment Tree！
```

```java
/**
 * 賠率歷史 Segment Tree
 * 支援：區間最大賠率查詢 O(log n)、賠率更新 O(log n)
 */
public class OddsSegmentTree {

    private final long[] maxTree;  // 儲存區間最大賠率（定點數，×100 後存 long）
    private final long[] minTree;  // 儲存區間最小賠率
    private final int n;

    public OddsSegmentTree(int capacity) {
        n = capacity;
        maxTree = new long[4 * n];
        minTree = new long[4 * n];
        Arrays.fill(maxTree, Long.MIN_VALUE);
        Arrays.fill(minTree, Long.MAX_VALUE);
    }

    /** 更新第 idx 個時間點的賠率（定點數） */
    public void update(int idx, long oddsCents, int node, int start, int end) {
        if (start == end) {
            maxTree[node] = oddsCents;
            minTree[node] = oddsCents;
            return;
        }
        int mid = (start + end) / 2;
        if (idx <= mid) update(idx, oddsCents, 2*node+1, start, mid);
        else            update(idx, oddsCents, 2*node+2, mid+1, end);
        maxTree[node] = Math.max(maxTree[2*node+1], maxTree[2*node+2]);
        minTree[node] = Math.min(minTree[2*node+1], minTree[2*node+2]);
    }

    /** 查詢 [l, r] 時間段內的最高賠率 */
    public long queryMax(int l, int r, int node, int start, int end) {
        if (r < start || end < l) return Long.MIN_VALUE;
        if (l <= start && end <= r) return maxTree[node];
        int mid = (start + end) / 2;
        return Math.max(
            queryMax(l, r, 2*node+1, start, mid),
            queryMax(l, r, 2*node+2, mid+1, end)
        );
    }

    // 便捷方法
    public void update(int idx, double odds) {
        update(idx, Math.round(odds * 100), 1, 0, n-1);
    }

    public double queryMaxOdds(int l, int r) {
        long result = queryMax(l, r, 1, 0, n-1);
        return result == Long.MIN_VALUE ? 0 : result / 100.0;
    }
}
```

## 三、賽事狀態機 — 有向圖 + FSM <!-- 💡 進階 -->

### 足球比賽狀態

```
賽事狀態轉換：

SCHEDULED（賽前）
  ↓ 開賽
FIRST_HALF（上半場）
  ↓ 進球 → GOAL_SCORED → FIRST_HALF
  ↓ 半場
HALF_TIME（中場休息，停止投注）
  ↓ 繼續
SECOND_HALF（下半場）
  ↓ 加時
EXTRA_TIME（延長賽，如有）
  ↓ 點球決勝
PENALTY_SHOOTOUT（點球）
  ↓
FINISHED（終場）
  ├─ 確認結果 → SETTLED（已結算）
  └─ 爭議 → DISPUTE → SETTLED 或 CANCELLED

投注截止規則：
  FIRST_HALF / SECOND_HALF：允許投注
  HALF_TIME / EXTRA_TIME / 以後：停止接受新投注
```

```java
public enum MatchState {
    SCHEDULED, FIRST_HALF, GOAL_SCORED, HALF_TIME,
    SECOND_HALF, EXTRA_TIME, PENALTY_SHOOTOUT,
    FINISHED, SETTLED, DISPUTE, CANCELLED;

    // 是否允許投注
    public boolean isBettingAllowed() {
        return this == FIRST_HALF || this == SECOND_HALF;
    }
}

@Service
public class MatchStateMachine {

    // 合法轉換：Map<當前狀態, 允許的下一個狀態集合>
    private static final Map<MatchState, Set<MatchState>> TRANSITIONS = Map.of(
        MatchState.SCHEDULED,        Set.of(MatchState.FIRST_HALF, MatchState.CANCELLED),
        MatchState.FIRST_HALF,       Set.of(MatchState.HALF_TIME, MatchState.GOAL_SCORED),
        MatchState.GOAL_SCORED,      Set.of(MatchState.FIRST_HALF, MatchState.SECOND_HALF),
        MatchState.HALF_TIME,        Set.of(MatchState.SECOND_HALF),
        MatchState.SECOND_HALF,      Set.of(MatchState.FINISHED, MatchState.EXTRA_TIME, MatchState.GOAL_SCORED),
        MatchState.EXTRA_TIME,       Set.of(MatchState.FINISHED, MatchState.PENALTY_SHOOTOUT),
        MatchState.PENALTY_SHOOTOUT, Set.of(MatchState.FINISHED),
        MatchState.FINISHED,         Set.of(MatchState.SETTLED, MatchState.DISPUTE),
        MatchState.DISPUTE,          Set.of(MatchState.SETTLED, MatchState.CANCELLED)
    );

    private final Map<Long, MatchState> matchStates = new ConcurrentHashMap<>();

    public void transition(Long matchId, MatchState newState) {
        MatchState current = matchStates.getOrDefault(matchId, MatchState.SCHEDULED);
        Set<MatchState> allowed = TRANSITIONS.getOrDefault(current, Set.of());

        if (!allowed.contains(newState)) {
            throw new IllegalStateException(
                String.format("比賽[%d]不能從 %s 轉換到 %s", matchId, current, newState));
        }

        matchStates.put(matchId, newState);

        // 狀態變更時，通知相關服務
        if (!newState.isBettingAllowed()) {
            bettingService.freezeBetting(matchId);  // 停止投注
        }
        if (newState == MatchState.SETTLED) {
            settlementService.settle(matchId);       // 觸發結算
        }
    }
}
```

## 四、高並發投注 — CAS + 樂觀鎖 <!-- 🔴 資深 -->

### 問題：萬人同時投「主隊贏」

```
Race Condition 場景：
  世足決賽，阿根廷 vs 法國
  100,000 人同時在最後 1 分鐘投「阿根廷贏」（賠率 2.5）

莊家風控：「阿根廷贏」這個選項最多承接 1,000,000 元投注
  超過上限就調低賠率（或暫停投注）

問題：如何在高並發下精確控制「已承接金額 < 上限」？
```

### 解法：Redis + Lua 原子操作

```java
@Service
public class BettingService {

    // Redis Key 設計：
    // "bet:pool:{matchId}:{optionId}"    → 該選項已承接投注總額
    // "bet:limit:{matchId}:{optionId}"   → 上限
    // "bet:odds:{matchId}:{optionId}"    → 當前賠率（定點數 ×100）

    private static final String BET_LUA = """
        local pool  = tonumber(redis.call('get', KEYS[1])) or 0
        local limit = tonumber(redis.call('get', KEYS[2])) or 0
        local amount = tonumber(ARGV[1])
        
        -- 風控：超過上限不允許投注
        if limit > 0 and pool + amount > limit then
            return -1  -- 超過限額
        end
        
        -- 原子累加
        redis.call('incrby', KEYS[1], amount)
        
        -- 記錄這筆投注（RPUSH 到 list）
        redis.call('rpush', KEYS[3], ARGV[2])  -- ARGV[2] = bet JSON
        
        return pool + amount  -- 返回更新後的累計金額
        """;

    public BetResult placeBet(BetRequest req) {
        // 風控前置檢查（快速失敗）
        if (!matchStateMachine.getState(req.getMatchId()).isBettingAllowed()) {
            return BetResult.BETTING_CLOSED;
        }

        String poolKey  = "bet:pool:"  + req.getMatchId() + ":" + req.getOptionId();
        String limitKey = "bet:limit:" + req.getMatchId() + ":" + req.getOptionId();
        String logKey   = "bet:log:"   + req.getMatchId() + ":" + req.getOptionId();

        String betJson = toJson(req);

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(BET_LUA, Long.class),
            List.of(poolKey, limitKey, logKey),
            String.valueOf(req.getAmount()), betJson
        );

        if (result == null || result < 0) {
            return BetResult.LIMIT_EXCEEDED;
        }

        // 非同步持久化到 DB（不阻塞投注請求）
        kafkaTemplate.send("bet-events", req);

        return BetResult.success(result);
    }
}
```

### 動態賠率調整：投注越多，賠率越低

```java
/**
 * Kelly Criterion 賠率調整（簡化版）
 * 原理：根據兩邊的投注比例動態調整賠率，確保莊家長期獲利
 *
 * 例如：主隊贏 投注 1,000,000 元，客隊贏 投注 200,000 元
 * 莊家給兩邊的賠率需要調整，讓無論哪邊贏，莊家都有利潤
 */
@Scheduled(fixedRate = 5000)  // 每 5 秒重新計算一次賠率
public void adjustOdds(Long matchId) {
    for (MatchOption option : getMatchOptions(matchId)) {
        long pool = getPool(matchId, option.getId());
        long totalPool = getTotalPool(matchId);

        if (totalPool == 0) continue;

        // 理論公允賠率 = 總投注 / 該選項投注
        double fairOdds = (double) totalPool / pool;

        // 莊家抽成（例如 5%）
        double adjustedOdds = fairOdds * 0.95;

        // 四捨五入到 0.05
        adjustedOdds = Math.round(adjustedOdds * 20.0) / 20.0;

        // 更新 Redis 中的賠率（WebSocket 推送給前端）
        updateOdds(matchId, option.getId(), adjustedOdds);
        webSocketService.broadcastOddsUpdate(matchId, option.getId(), adjustedOdds);
    }
}
```

## 五、結算系統 — 優先佇列 + 分批處理 <!-- 💡 進階 -->

### 比賽結束後的結算流程

```
終場 → 觸發結算任務

結算步驟：
  1. 確認最終比分
  2. 計算每個中獎投注的獎金 = 投注金額 × 賠率
  3. 按用戶 ID 彙整（同一用戶可能投了多注）
  4. 批次打款到用戶帳戶（不能一筆一筆打，太慢）
  5. 記錄結算日誌（合規要求）

難點：
  一場熱門比賽可能有 100,000 筆投注需要結算
  不能一次全部 SELECT 到記憶體（OOM）
  不能逐筆打款（太多 DB 交易）
```

```java
@Service
public class SettlementService {

    /** 分批結算，每批 500 筆 */
    public void settle(Long matchId) {
        MatchResult result = getMatchResult(matchId);
        Set<Long> winningOptions = result.getWinningOptionIds();

        // 用優先佇列按用戶 ID 排序（方便合併同一用戶的多筆中獎）
        PriorityQueue<Settlement> pq =
            new PriorityQueue<>(Comparator.comparingLong(s -> s.getUserId()));

        // 游標分頁（避免一次全部載入）
        long lastBetId = 0;
        int batchSize  = 500;

        do {
            List<Bet> bets = betRepository.findWinningBets(
                matchId, winningOptions, lastBetId, batchSize);

            if (bets.isEmpty()) break;

            for (Bet bet : bets) {
                long reward = Math.round(bet.getAmount() * bet.getOdds());  // 四捨五入
                pq.offer(new Settlement(bet.getUserId(), reward, bet.getId()));
            }

            // 合併同一用戶的多筆獎金，批次打款
            processBatch(pq, bets.size() < batchSize);

            lastBetId = bets.get(bets.size() - 1).getId();
        } while (true);
    }

    private void processBatch(PriorityQueue<Settlement> pq, boolean isLast) {
        if (!isLast && pq.size() < 100) return;  // 攢夠 100 筆再處理

        Map<Long, Long> userRewards = new HashMap<>();
        while (!pq.isEmpty()) {
            Settlement s = pq.poll();
            userRewards.merge(s.getUserId(), s.getAmount(), Long::sum);
        }

        // 批次打款（一次 SQL INSERT 多條，減少 DB 交易次數）
        accountService.batchCredit(userRewards);
    }
}
```

## 六、體育彩票資料結構選型速查

| 功能 | 資料結構 | 理由 |
|------|----------|------|
| 賠率歷史查詢（區間最值） | Segment Tree | O(log n) 查詢/更新 |
| 賽事狀態管理 | 有限狀態機（FSM） | 防止非法狀態轉換 |
| 高並發投注控制 | Redis Lua + 原子計數 | 無鎖，高吞吐 |
| 動態賠率計算 | 定時任務 + 定點數運算 | 精確，不用浮點數 |
| 結算分批處理 | 優先佇列 + 游標分頁 | 合併同用戶，避免 OOM |
| 用戶投注歷史 | 倒排索引（按用戶/按比賽） | 雙向查詢 |
| 賠率即時推送 | Redis Pub/Sub + WebSocket | 多節點廣播，低延遲 |

## 七、合規稽核設計 <!-- 🔴 資深 -->

```java
/**
 * 所有投注、賠率變化、結算都要記錄到不可竄改的日誌
 * 使用 Append-Only Log（只能追加，不能修改/刪除）
 */
@Service
public class AuditLogService {

    // 日誌格式：每條記錄含 hash（前一條記錄的 hash），形成區塊鏈式結構
    // 任何人修改中間的記錄，後面所有記錄的 hash 都會不對 → 可偵測篡改

    public AuditEntry append(AuditEntry entry) {
        String prevHash = getLastHash();  // 取上一條的 hash

        // 計算這條記錄的 hash（含上一條 hash）
        String content = entry.toJson() + prevHash;
        String hash = sha256(content);
        entry.setHash(hash);
        entry.setPrevHash(prevHash);

        auditRepository.save(entry);  // 只插入，永遠不更新/刪除
        return entry;
    }

    /** 驗證日誌完整性（監管稽核時使用） */
    public boolean verify() {
        List<AuditEntry> entries = auditRepository.findAllOrderById();
        for (int i = 1; i < entries.size(); i++) {
            AuditEntry curr = entries.get(i);
            AuditEntry prev = entries.get(i-1);

            // 驗證前後連結
            if (!curr.getPrevHash().equals(prev.getHash())) {
                return false;  // 有記錄被竄改！
            }

            // 驗證 hash 計算正確
            String expected = sha256(curr.toJson() + curr.getPrevHash());
            if (!expected.equals(curr.getHash())) {
                return false;
            }
        }
        return true;
    }
}
```
